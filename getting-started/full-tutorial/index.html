    Getting Started with DRUIDS¶

    • TODO: Proof read this
    • TODO: Format correctly

    This tutorial walks you through setting up DRUIDS on your local machine.

    Prerequisites¶

    • Git installed
    • GitHub account with access to ks-sbc organization
    • Obsidian (free note-taking app)
    • Node.js 22+ (for website development)

    Step 1: Clone the Repositories¶

    # Create a workspace
mkdir -p ~/ksbc/repos
cd ~/ksbc/repos

# Clone the main repositories
git clone https://github.com/ks-sbc/druids-wiki.git
git clone https://github.com/ks-sbc/.github.git
git clone https://github.com/ks-sbc/shared.git
git clone https://github.com/ks-sbc/website.git

# Clone vault repositories based on your access level
git clone https://github.com/ks-sbc/vault-public.git vault/public
git clone https://github.com/ks-sbc/vault-member.git vault/member
# Only if you have L2 access:
# git clone https://github.com/ks-sbc/vault-cadre.git vault/cadre

    Step 2: Set Up Obsidian¶

    1. Open Obsidian
    2. Create a new vault pointing to ~/ksbc/repos/vault/public
    3. The shared configuration will be automatically linked via symlinks

    Step 3: Verify Your Setup¶

    Run these commands to verify everything is working:

    # Check symlinks
ls -la vault/public/.obsidian
# Should show: .obsidian -> /path/to/shared/obsidian

# Check Node.js version (for website work)
node --version
# Should show: v22.x.x or higher

    Step 4: Test the Meeting Minutes Workflow¶

    cd .github
# Install act for local testing
# See: https://github.com/nektos/act#installation
act -W .github/workflows/meeting-minutes.yml
