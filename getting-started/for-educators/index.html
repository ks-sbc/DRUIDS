    Your Journey: From Understanding to Teaching¶

    This guide equips you to teach DRUIDS effectively - whether running a 2-hour security workshop, facilitating organizational adoption, or building a culture of democratic knowledge management. No technical expertise required, just a commitment to empowering others.

    Chapter 1: Understanding Your Role¶

    As an Educator, You Are¶

    1. Security Advocate - Making security intuitive, not intimidating
    2. Culture Builder - Shifting from "need to know" to "need to share safely"
    3. Technical Translator - Explaining complex concepts simply
    4. Change Facilitator - Guiding organizational transformation
    5. Practice Leader - Modeling good security habits

    Your Teaching Toolkit¶

    Core Concepts to Master¶

    • Three-tier security model (visual, not technical)
    • Democratic centralism in practice
    • Git as organizational memory
    • Security as collective responsibility

    Teaching Methods¶

    • Hands-on demonstrations
    • Real-world scenarios
    • Visual learning aids
    • Peer teaching
    • Incremental skill building

    Chapter 2: Workshop Frameworks¶

    2.1 The 2-Hour Security Awareness Workshop¶

    Audience: All members, no technical background assumed
    Goal: Build security intuition and basic practices

    Agenda (120 minutes)¶

    0:00-0:15 - The Problem

    Activity: "Security Horror Stories"
- Share real examples (anonymized)
- Calculate organizational risk
- Introduce "One Person Has All Passwords" problem
Materials: Flipchart, risk calculator worksheet

    0:15-0:35 - Visual Security Demo

    Activity: "See Security"
- Show three documents (green/brown/red borders)
- Sort exercise with printed examples
- Explain what belongs where
Materials: Sample documents, sorting cards

    0:35-0:55 - Democratic Workflow

    Activity: "Proposal to Implementation"
- Walk through real proposal
- Show discussion  decision  action
- Emphasize transparency with security
Materials: Example proposal, workflow diagram

    0:55-1:15 - Hands-On Practice

    Activity: "Create Your First Secure Note"
- Open Obsidian together
- Create meeting minutes
- Classify security level
- See visual indicators work
Materials: Computers/tablets, practice vault

    1:15-1:35 - Daily Workflows

    Activity: "A Day in DRUIDS"
- Morning: Pull latest changes
- Meeting: Take secure notes
- Tasks: Track assignments
- Evening: Commit and push
Materials: Daily checklist cards

    1:35-1:50 - Q&A and Troubleshooting

    Activity: "Stump the Trainer"
- Address specific concerns
- Work through scenarios
- Build confidence
Materials: FAQ sheet, scenario cards

    1:50-2:00 - Commitment and Next Steps

    Activity: "Security Pledge"
- Personal security commitments
- Buddy system setup
- Schedule follow-up
Materials: Commitment cards, buddy assignments

    2.2 The Half-Day Implementation Workshop¶

    Audience: Implementation team (3-5 people)
    Goal: Set up DRUIDS infrastructure

    Agenda (4 hours)¶

    Hour 1: Technical Setup

    • GitHub organization creation
    • Three-repository structure
    • Obsidian installation
    • First sync test

    Hour 2: Configuration

    • Security indicators setup
    • Plugin configuration
    • Template creation
    • Workflow customization

    Hour 3: Content Migration

    • Document inventory
    • Security classification
    • Import process
    • Verification checks

    Hour 4: Team Training

    • Role assignments
    • Practice workflows
    • Troubleshooting
    • Launch planning

    2.3 The Weekly Skill-Building Series¶

    Format: 6 sessions, 1 hour each
    Goal: Deep expertise development

    Session Plan¶

    Week 1: Security Fundamentals

    • Three-tier model deep dive
    • Threat modeling basics
    • Security habit formation

    Week 2: Git Essentials

    • Version control concepts
    • Branch workflows
    • Conflict resolution

    Week 3: Advanced Obsidian

    • Plugin ecosystem
    • Automation setup
    • Custom workflows

    Week 4: Democratic Processes

    • Proposal templates
    • Voting mechanisms
    • Decision documentation

    Week 5: Troubleshooting

    • Common problems
    • Debug techniques
    • Getting help

    Week 6: Teaching Others

    • Train-the-trainer
    • Creating materials
    • Building culture

    Chapter 3: Teaching Techniques¶

    3.1 Making Security Intuitive¶

    The Traffic Light Method¶

     Green = Go (share freely)
 Yellow = Yield (members only)  
 Red = Stop (restricted access)

    Teaching Script: "Imagine each document as having a traffic light. Green means anyone can see it - like a public flyer. Yellow means slow down - only members should see this, like meeting minutes. Red means stop - highly restricted, like security plans."

    The House Metaphor¶

     Your Organization's House:
- Front yard (L0) = What neighbors see
- Living room (L1) = Where members gather
- Safe room (L2) = Most protected space

    Activity: Draw the house, place different documents in appropriate rooms.

    3.2 Explaining Git Simply¶

    The Photo Album Analogy¶

    "Git is like a photo album that never forgets. Every time you save, it takes a snapshot. You can always go back and see what things looked like last week, last month, or last year."

    Key Concepts Simplified¶

    • Commit = Taking a snapshot
    • Push = Putting album in shared library
    • Pull = Getting latest album version
    • Branch = Making a draft version

    3.3 Democratic Centralism in Practice¶

    The Potluck Model¶

    "Democratic centralism is like organizing a potluck:

    1. Proposal: 'Let's have a potluck'
    2. Discussion: 'What kind? When? Where?'
    3. Decision: 'Vegetarian potluck, Saturday, 6pm'
    4. Unity: Everyone brings vegetarian dishes
    5. Review: 'How did it go? Improvements?'"

    Chapter 4: Building Culture¶

    4.1 Security Champions Program¶

    Identifying Champions¶

    • Natural teachers in your organization
    • Security-conscious members
    • Early adopters of new tools
    • Bridge between technical and non-technical

    Champion Responsibilities¶

    1. First point of contact for questions
    2. Weekly security check-ins
    3. Spot training as needed
    4. Feedback to educators

    Supporting Champions¶

    • Monthly champion meetings
    • Advanced training opportunities
    • Recognition in meetings
    • Small troubleshooting budget

    4.2 Incremental Adoption Strategy¶

    Phase 1: Foundation (Weeks 1-2)¶

    • Leadership buy-in
    • Core team training
    • Basic infrastructure
    • First successful sync

    Phase 2: Early Adoption (Weeks 3-4)¶

    • Daily meeting notes
    • Task tracking
    • Security classification
    • Visual indicators working

    Phase 3: Expansion (Weeks 5-8)¶

    • All members onboarded
    • Workflows customized
    • Problems resolved
    • Habits forming

    Phase 4: Maturity (Months 3+)¶

    • Self-sustaining system
    • Peer teaching
    • Continuous improvement
    • Cultural integration

    4.3 Overcoming Resistance¶

    Common Objections and Responses¶

    "It's too technical"

    "You use email and web browsers daily. DRUIDS is actually simpler - it just looks different at first."

    "We don't need this level of security"

    "Every organization thinks that until something happens. This is insurance you hope never to need."

    "It will slow us down"

    "Initially, yes. Within a month, you'll save hours weekly on finding information and onboarding people."

    "What if we mess it up?"

    "Git remembers everything. We can always go back. It's actually harder to lose data than before."

    Chapter 5: Workshop Materials¶

    5.1 Pre-Workshop Checklist¶

    Two Weeks Before¶

    • Survey participants for technical comfort
    • Prepare example documents
    • Test all equipment
    • Create practice vault
    • Print handouts

    One Week Before¶

    • Send prep materials
    • Confirm attendance
    • Review security scenarios
    • Prepare name tags with security levels
    • Charge all devices

    Day Before¶

    • Set up room
    • Test projector/screen
    • Prepare flip charts
    • Review participant needs
    • Practice demonstrations

    5.2 Essential Handouts¶

    Security Quick Reference Card¶

  Educational materials             
  Public statements                 
  General resources                 

  L1 MEMBER                        
  Meeting minutes                   
  Internal plans                    
  Member contacts

    Daily Workflow Checklist¶

    Morning (2 min)
 Open Obsidian
 Pull latest changes
 Review assigned tasks

During Work
 Create notes with security level
 Check visual indicators
 Save regularly

End of Day (3 min)
 Review changes
 Commit with message
 Push to repository

    5.3 Interactive Exercises¶

    Exercise 1: Security Sorting Game¶

    Materials: 20 document cards, 3 boxes (green/brown/red) Time: 15 minutes Process:

    1. Divide into small groups
    2. Sort documents into appropriate boxes
    3. Discuss disagreements
    4. Review correct answers

    Sample Documents:

    • "Public Education Flyer" L0
    • "Meeting Minutes 3/15" L1
    • "Security Protocol Review" L2
    • "Newsletter Draft" L0
    • "Member Phone List" L1

    Exercise 2: Workflow Race¶

    Materials: Printed workflow steps, timer Time: 10 minutes Process:

    1. Teams arrange steps in correct order
    2. First team done calls time
    3. Review and discuss

    Exercise 3: Security Scenario Response¶

    Materials: Scenario cards Time: 20 minutes Process:

    1. Read scenario aloud
    2. Groups discuss response
    3. Share approaches
    4. Provide best practice

    Sample Scenarios:

    • "A journalist wants our meeting minutes"
    • "New member needs access to documents"
    • "Someone accidentally commits L2 to L0"
    • "Computer stolen with vault open"

    Chapter 6: Assessment and Certification¶

    6.1 Skill Assessment Framework¶

    Basic Certification (All Members)¶

    • Can identify security levels visually
    • Creates properly classified documents
    • Completes daily workflow independently
    • Responds correctly to security scenarios

    Advanced Certification (Champions)¶

    • Troubleshoots common problems
    • Teaches basic skills to others
    • Manages Git conflicts
    • Customizes workflows

    Trainer Certification¶

    • Delivers all three workshop types
    • Adapts to different audiences
    • Creates custom materials
    • Measures learning outcomes

    6.2 Learning Metrics¶

    Immediate (End of Workshop)¶

    • Security classification accuracy: ___%
    • Workflow step recall: ___%
    • Confidence level: __/10
    • Likelihood to use: __/10

    30-Day Follow-Up¶

    • Daily use rate: ___%
    • Problems encountered: ___
    • Peer teaching instances: ___
    • Security incidents: ___

    90-Day Assessment¶

    • Workflow integration: Complete/Partial/None
    • Cultural adoption: High/Medium/Low
    • Time savings: ___ hours/week
    • Knowledge retention: ___%

    Chapter 7: Troubleshooting Teaching Challenges¶

    7.1 Common Teaching Challenges¶

    "The Tech-Phobic Participant"¶

    Signs: Arms crossed, "I'm not good with computers" Approach:

    • Pair with tech-comfortable buddy
    • Start with paper exercises
    • Celebrate small wins
    • Share your own struggles

    "The Know-It-All"¶

    Signs: Interrupts, corrects others, shows off Approach:

    • Channel into helper role
    • Ask to assist others
    • Give advanced challenges
    • Acknowledge expertise

    "The Skeptic"¶

    Signs: Questions everything, resists change Approach:

    • Welcome questions
    • Provide evidence/examples
    • Connect to their values
    • Give time to process

    7.2 Technical Issues During Training¶

    Internet Fails¶

    Backup Plan:

    • Offline demo vault prepared
    • Screenshots of key processes
    • Focus on concepts, not clicking

    Equipment Problems¶

    Backup Plan:

    • Spare devices available
    • Pair/share protocols
    • Projection alternatives

    Software Won't Install¶

    Backup Plan:

    • Pre-installed options
    • USB with installers
    • Web-based alternatives

    7.3 Organizational Resistance¶

    Leadership Not Bought In¶

    • Start with security risks
    • Show time/cost savings
    • Offer pilot program
    • Measure and report

    Competing Priorities¶

    • Integrate with existing meetings
    • Micro-learning approach
    • Show immediate value
    • Connect to mission

    Chapter 8: Advanced Teaching Topics¶

    8.1 Accessibility Considerations¶

    Visual Impairments¶

    • High contrast themes
    • Screen reader compatibility
    • Verbal descriptions of visuals
    • Tactile security indicators

    Motor Limitations¶

    • Keyboard-only navigation
    • Voice control options
    • Customized shortcuts
    • Flexible timing

    Cognitive Differences¶

    • Multiple learning modalities
    • Repeat key concepts
    • Visual aids and diagrams
    • Peer support system

    8.2 Multi-Language Organizations¶

    Translation Strategy¶

    • Prioritize UI elements
    • Create multilingual templates
    • Document in primary language
    • Visual instructions when possible

    Cultural Adaptations¶

    • Security metaphors that translate
    • Respectful of hierarchies
    • Collaborative approaches
    • Local examples

    8.3 Remote Training Adaptations¶

    Virtual Workshop Setup¶

    • Pre-session tech check
    • Breakout rooms for practice
    • Screen sharing protocols
    • Recording permissions

    Engagement Techniques¶

    • Polls and quizzes
    • Chat participation
    • Virtual backgrounds
    • Collaborative documents

    Quick Reference: Teaching DRUIDS¶

    
         EDUCATOR'S QUICK GUIDE          

 Core Message:                           
 "Security through simplicity"           

 Teaching Order:                         
 1. Why (problem/solution)               
 2. What (three-tier model)              
 3. How (hands-on practice)              
 4. Daily (workflow habits)              

 Key Analogies:                          
  Traffic lights for security           
  Photo album for Git                   
  House rooms for access levels         

 Success Metrics:                        
  90% classify correctly                
  80% complete daily workflow           
  70% help others
