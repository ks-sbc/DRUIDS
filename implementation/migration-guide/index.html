    Migration Guide: From Google Docs to DRUIDS¶

    Most organizations complete migration in one weekend. This guide shows you how, without losing data or disrupting work.

    Before You Start: Reality Check¶

    You're Ready to Migrate If¶

    Someone burned out from password management
    Can't find last month's meeting notes
    Worried about Google reading everything
    New members confused about where things are
    Just tired of the chaos

    You're NOT Ready If¶

    No one sees the current system as broken
    Can't dedicate one weekend to migration
    No technical champion identified
    Active crisis requiring all hands

    Phase 1: Pre-Migration Audit (Friday Evening, 2 hours)¶

    Step 1.1: Document Inventory Checklist¶

    Create a spreadsheet listing:

    Document Name | Location | Last Updated | Security Level | Owner
-----------|----------|--------------|----------------|-------
Meeting Minutes 2024 | Sarah's Drive | Nov 2024 | L1 | Sarah
Budget 2024 | Shared Drive | Oct 2024 | L1 | Treasury
Flyer Templates | Mike's Drive | Sep 2024 | L0 | Outreach
Action Plans | Signal chat | ??? | L2 | Various

    Common Locations to Check:

    • Personal Google Drives (each member)
    • Shared Drives
    • Google Groups
    • Email attachments
    • Signal/Discord pinned messages
    • That one person's laptop

    Step 1.2: Quick Security Classification¶

    For each document, ask:

    1. Is it already public? L0 (Green)
    2. Would exposure harm org but not people? L1 (Brown)
    3. Could someone get arrested/fired? L2 (Red)

    Classification Shortcuts:

    • Meeting agendas L0
    • Meeting minutes (full) L1
    • Member lists L1
    • Financial records L1
    • Action planning L2
    • Legal defense L2

    Step 1.3: Team Readiness Check¶

    Technical Champion Checklist:

    • Comfortable with basic Git (or willing to learn)
    • Can dedicate weekend to migration
    • Patient with non-technical members
    • Has admin access to Google accounts

    Team Communication:

    Subject: Document Migration This Weekend

Comrades,

We're moving from Google Docs to DRUIDS this weekend to solve our 
"one person has all passwords" problem. 

What you need to do:
1. List any important docs you have by Friday night
2. Be available Saturday for questions
3. Join training session Sunday 2pm

This will make everyone's life easier - promise!

[Technical Champion Name]

    Phase 2: Export Process (Saturday Morning, 3 hours)¶

    Step 2.1: Bulk Export from Google¶

    Using Google Takeout (Recommended):

    1. Go to takeout.google.com
    2. Deselect all, then select only "Drive"
    3. Choose export format: .docx for docs, .xlsx for sheets
    4. Export frequency: One-time
    5. File type: .zip, 2GB segments
    6. Wait for email (usually 1-2 hours)

    Manual Export (For Specific Folders):

    1. Open Google Drive
    2. Right-click folder Download
    3. Converts to Office formats automatically

    Step 2.2: Convert to Markdown¶

    Automated Conversion:

    # Install pandoc (one-time)
# Mac: brew install pandoc
# Linux: sudo apt install pandoc
# Windows: Download from pandoc.org

# Convert all docs to markdown
find ./GoogleExport -name "*.docx" -exec \
  pandoc -f docx -t markdown {} -o {}.md \;

    Manual Conversion Options:

    • Docs to Markdown add-on
    • Copy-paste into Obsidian (handles formatting)
    • MarkdownConverter.com for one-offs

    Step 2.3: Organize by Security Tier¶

    Create folder structure:

    migration-staging/
 L0-public/
    education/
    flyers/
    public-statements/
 L1-member/
    meeting-minutes/
    financials/
    internal-docs/
 L2-cadre/
     action-planning/
     legal/

    Quick Sort Script:

    # Move files based on keywords (customize for your org)
grep -l "meeting minutes" *.md | xargs -I {} mv {} ./L1-member/meeting-minutes/
grep -l "public" *.md | xargs -I {} mv {} ./L0-public/
grep -l "action\|legal\|security" *.md | xargs -I {} mv {} ./L2-cadre/

    Phase 3: Import to DRUIDS (Saturday Afternoon, 3 hours)¶

    Step 3.1: Vault Setup¶

    # Clone the vaults (adjust URLs for your org)
git clone https://github.com/your-org/vault-public.git
git clone https://github.com/your-org/vault-member.git
git clone https://github.com/your-org/vault-cadre.git

# Copy sorted documents
cp -r migration-staging/L0-public/* vault-public/
cp -r migration-staging/L1-member/* vault-member/
cp -r migration-staging/L2-cadre/* vault-cadre/

    Step 3.2: Add DRUIDS Metadata¶

    Quick Metadata Template:

    ---
title: "Original Document Title"
document_id: "IMPORT-2025-001-L1"
security: "L1"
version: "1.0.0"
migrated_from: "Google Docs"
original_date: "2024-11-15"
tags: ["imported", "meeting-minutes"]
---

    Step 3.3: Commit to Git¶

    # For each vault
cd vault-public
git add .
git commit -m "Initial migration from Google Docs"
git push

# Repeat for member and cadre vaults

    Phase 4: Parallel Run (One Week)¶

    Week Schedule¶

    Monday: Soft Launch

    • Core team uses DRUIDS for new documents
    • Google Docs remains primary
    • Note any issues

    Tuesday-Thursday: Gradual Adoption

    • Daily check-ins (15 min)
    • Fix formatting issues
    • Add missing documents
    • Train struggling members

    Friday: Decision Point

    • Team vote: Ready for full cutover?
    • If no: One more week parallel run
    • If yes: Weekend cutover

    Common Issues & Fixes¶

    Issue Fix
    "Can't find anything" Create index pages with links
    "Formatting is broken" Usually table conversion - fix manually
    "Miss real-time collab" Schedule co-working sessions
    "Too many passwords" Use password manager, share vault
    "Git is confusing" Pair with technical champion

    Phase 5: Cutover (Following Weekend)¶

    Step 5.1: Final Sync¶

    # One last export of any new Google Docs
# Check for docs created during parallel run
# Import using same process

    Step 5.2: Archive Google Docs¶

    Don't Delete! Archive Instead:

    1. Create folder "ARCHIVED - Moved to DRUIDS"
    2. Move all migrated docs there
    3. Remove edit permissions (view only)
    4. Add README: "We now use DRUIDS - see [link]"

    Step 5.3: Update Documentation¶

    • Discord/Signal: Pin DRUIDS links
    • Email signatures: Add vault URLs
    • Meeting templates: Update to DRUIDS format
    • Bookmark key pages

    Step 5.4: Celebration Checklist¶

    • Team message: "We did it!"
    • Share search demo (find last year's decision in 5 seconds)
    • Highlight security improvements
    • Thank technical champion
    • Schedule 2-week retro

    Post-Migration Support¶

    First Month Schedule¶

    Week 1: Daily Check-ins

    • 15-min standups
    • Fix immediate blockers
    • Celebrate small wins

    Week 2: Refine Workflows

    • Optimize folder structures
    • Create custom templates
    • Set up automation

    Week 3: Advanced Features

    • Dataview dashboards
    • Meeting minute automation
    • Security tier training

    Week 4: Full Retro

    • What worked?
    • What needs improvement?
    • Ready to help another org migrate?

    Success Metrics¶

    Track these to show value:

    • Time to find last month's decision: 2 min 10 sec
    • Meeting notes completion rate: 60% 95%
    • New member onboarding: 2 months 1 week
    • Security incidents: Track prevented leaks

    Emergency Rollback Plan¶

    If everything goes wrong:

    1. Google Docs are archived, not deleted
    2. Re-enable edit permissions
    3. Do proper retro on what failed
    4. Try again with lessons learned

    Remember: Even "failed" migrations teach valuable lessons. The attempt alone often highlights organizational issues worth fixing.
