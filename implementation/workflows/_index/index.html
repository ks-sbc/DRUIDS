    Workflow Guides¶

    This section contains practical guides for common workflows in DRUIDS. Each guide provides step-by-step instructions for specific organizational tasks.

    Core Workflows¶

    Proposal Process¶

    Complete workflow for submitting, discussing, and implementing proposals using democratic centralist principles.

    Meeting Workflow Guide¶

    Standardized process for conducting meetings and recording decisions.

    Project Management Guide¶

    Managing projects from inception through completion with accountability.

    Additional Resources¶

    For member onboarding and security implementation, see:

    • Security Model - Three-tier security model
    • Git Workflow Guide - Version control processes

    Daily Operations¶

    Creating Documents¶

    1. Use templates for consistency
    2. Add proper metadata/frontmatter
    3. Place in correct folder
    4. Link to related documents
    5. Set appropriate security level

    Finding Information¶

    1. Use search (Ctrl/Cmd + Shift + F)
    2. Browse folder structure
    3. Follow links between documents
    4. Use Dataview queries
    5. Check dashboards

    Updating Status¶

    1. Locate relevant document
    2. Update frontmatter fields
    3. Add progress notes
    4. Update linked documents
    5. Commit changes (if using Git)

    Workflow Principles¶

    Consistency¶

    • Use templates for all document types
    • Follow naming conventions
    • Maintain folder structure
    • Apply metadata consistently

    Transparency¶

    • Document all decisions
    • Link related information
    • Update status regularly
    • Make information findable

    Security¶

    • Classify documents appropriately
    • Respect access levels
    • Follow compartmentalization
    • Maintain need-to-know

    Accountability¶

    • Assign clear ownership
    • Set concrete deadlines
    • Track progress objectively
    • Report regularly

    Common Patterns¶

    Theory â†’ Practice Cycle¶

    1. Study theoretical materials
    2. Develop implementation plan
    3. Execute in practice
    4. Document results
    5. Extract lessons learned
    6. Refine theory

    Decision Implementation¶

    1. Record decision in minutes
    2. Create action items
    3. Assign responsibilities
    4. Track progress
    5. Report completion
    6. Evaluate effectiveness

    Problem Resolution¶

    1. Document issue clearly
    2. Gather relevant information
    3. Propose solutions
    4. Discuss democratically
    5. Implement decision
    6. Monitor results

    Integration Points¶

    With Git¶

    • Commit after significant changes
    • Use meaningful commit messages
    • Create branches for proposals
    • Merge after approval

    With Dashboards¶

    • Update metadata for queries
    • Check dashboards regularly
    • Use metrics for decisions
    • Maintain data quality

    With Security¶

    • Apply classification immediately
    • Update access as needed
    • Audit regularly
    • Train on security workflows

    Best Practices¶

    Daily Habits¶

    1. Review dashboard each morning
    2. Update task progress
    3. Process inbox folder
    4. Link new information
    5. Commit changes

    Weekly Routines¶

    1. Review all active projects
    2. Update member records
    3. Archive completed items
    4. Plan upcoming work
    5. Run security audit

    Monthly Processes¶

    1. Generate reports
    2. Conduct criticism/self-criticism
    3. Review and refine workflows
    4. Update documentation
    5. Train new members

    Troubleshooting Workflows¶

    "Can't find information"¶

    • Check search syntax
    • Verify folder location
    • Review recent changes
    • Check links from related docs

    "Workflow too complex"¶

    • Start with basic version
    • Add steps gradually
    • Document pain points
    • Iterate improvements

    "Not being followed"¶

    • Simplify process
    • Increase training
    • Add accountability
    • Review necessity

    Customizing for Your Organization¶

    1. Assess Needs: What are your pain points?
    2. Start Simple: Basic workflows first
    3. Document Changes: Track what you modify
    4. Train Members: Ensure understanding
    5. Iterate: Improve based on usage

    Getting Started¶

    New to DRUIDS workflows? Follow this sequence:

    1. Read Security Model first
    2. Practice with test documents
    3. Learn Meeting Workflow Guide process
    4. Try a simple Proposal Process
    5. Implement Project Management Guide

    Quick Reference¶

    Essential Shortcuts:

    • New note: Ctrl/Cmd + N
    • Search: Ctrl/Cmd + Shift + F
    • Quick switcher: Ctrl/Cmd + O
    • Command palette: Ctrl/Cmd + P

    Key Principles:

    • Document everything important
    • Link related information
    • Update status regularly
    • Follow security protocols
    • Maintain consistency

    Remember: Workflows should serve your organizing, not complicate it. Start simple and evolve based on real needs.
