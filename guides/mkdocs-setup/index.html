    MkDocs Setup and Usage Guide¶

    Overview¶

    This tutorial walks you through setting up and using MkDocs to build the DRUIDS documentation website. You'll learn how to install MkDocs, configure it for our project, and build/deploy the documentation site.

    Key Learning: This tutorial reflects real-world experience from implementing DRUIDS documentation. We'll show you both the ideal path and common pitfalls to avoid.

    Success Pattern: Start with Material theme + custom CSS rather than building custom themes from scratch. This approach delivers professional results in minutes rather than hours of debugging.

    Prerequisites¶

    • Python 3.8 or higher
    • pip (Python package manager)
    • Git
    • Text editor

    Installation¶

    1. Install MkDocs and Dependencies¶

    Recommended Approach (Battle-tested for DRUIDS):

    # Create a virtual environment (highly recommended)
python -m venv mkdocs-env
source mkdocs-env/bin/activate  # On Windows: mkdocs-env\Scripts\activate

# Install Material theme (includes MkDocs + all dependencies)
pip install mkdocs-material

# Optional: Additional plugins for enhanced features
pip install mkdocs-git-revision-date-localized-plugin

# Verify installation
mkdocs --version
# Should show: mkdocs, version 1.6.1+

    Why Material theme first?

    • Installing mkdocs-material automatically includes MkDocs core
    • Includes all required pymdownx extensions
    • Avoids dependency conflicts that cause "cannot find module" errors
    • Provides proven, working navigation (no dropdown issues)

    Common Installation Issues:

    # Error: "cannot find module 'pymdownx'"
# Solution: Install material theme (includes pymdownx)
pip install mkdocs-material

# Error: "cannot find module 'materialx.emoji'"
# This indicates wrong emoji configuration - we'll fix in config section

    2. Clone the DRUIDS Documentation¶

    git clone https://github.com/ks-sbc/druids-wiki.git
cd druids-wiki

    Basic Usage¶

    1. Start the Development Server¶

    Basic Local Development:

    # Navigate to the project directory
cd druids-wiki

# Start the development server
mkdocs serve

# You should see:
# INFO    -  Building documentation...
# INFO    -  Cleaning site directory
# INFO    -  Documentation built in 0.45 seconds
# INFO    -  [10:30:45] Serving on http://127.0.0.1:8000/

    For Team Development (DRUIDS Recommended):

    # External access for team collaboration
mkdocs serve --dev-addr=0.0.0.0:8001

# Now accessible at:
# - http://localhost:8001 (your machine)
# - http://YOUR_IP:8001 (other team members)

    Development Server Features:

    • 🔄 Live reload - Changes appear instantly
    • 🚀 Fast builds - Rebuilds only changed files
    • ⚠️ Link validation - Shows broken links as warnings
    • 🔍 Search indexing - Updates search automatically

    Open your browser to see the documentation site with working navigation!

    2. Make Changes and See Live Updates¶

    1. Edit a documentation file (e.g., docs/index.md)
    2. Save the file
    3. Watch the browser automatically refresh with your changes

    This live-reload feature makes it easy to write and preview documentation simultaneously.

    3. Build the Static Site¶

    # Build production-ready site
mkdocs build

# This creates a 'site' directory with HTML files
ls site/
# You'll see: index.html, assets/, stylesheets/, etc.

    Configuration¶

    Understanding mkdocs.yml¶

    The mkdocs.yml file in the project root controls how your documentation is built.

    DRUIDS Production Configuration (Battle-tested):

    # Basic site information
site_name: DRUIDS Wiki
site_description: Kansas Socialist Book Club's documentation for DRUIDS

# Essential plugins (minimal, reliable set)
plugins:
  - search    # Built-in search functionality

# Custom CSS for DRUIDS revolutionary aesthetics
extra_css:
  - stylesheets/druids-material.css

# Theme configuration (proven to work)
theme:
  name: material
  features:
    - navigation.tabs         # Top-level tabs (fixes dropdown issues)
    - navigation.sections     # Section grouping
    - navigation.expand       # Auto-expand navigation
    - navigation.path         # Breadcrumb navigation
    - navigation.top          # Back to top button
    - search.highlight        # Highlight search terms
    - search.share           # Shareable search URLs
    - content.code.copy      # Copy buttons for code blocks

  # DRUIDS revolutionary color scheme
  palette:
    - scheme: slate           # Dark theme (primary)
      primary: green          # Forest green
      accent: amber           # Earth tone accent
      toggle:
        icon: material/brightness-4
        name: Switch to light mode
    - scheme: default         # Light theme (fallback)
      primary: green
      accent: amber
      toggle:
        icon: material/brightness-7
        name: Switch to dark mode

  font:
    text: Inter             # Clean, readable font
    code: JetBrains Mono    # Monospace for code

# Markdown extensions (essential set)
markdown_extensions:
  - abbr              # Abbreviation definitions
  - admonition        # Call-out boxes
  - attr_list         # HTML attributes in Markdown
  - def_list          # Definition lists
  - footnotes         # Footnote references
  - md_in_html        # Markdown inside HTML
  - toc:              # Table of contents
      permalink: true # Anchor links for headings
  - pymdownx.superfences:   # Enhanced code blocks
      custom_fences:
        - name: mermaid     # Diagram support
          class: mermaid
          format: !!python/name:pymdownx.superfences.fence_code_format
  - pymdownx.highlight      # Syntax highlighting
  - pymdownx.details       # Collapsible sections
  - pymdownx.tasklist:     # Task lists
      custom_checkbox: true
  - pymdownx.emoji:        # Emoji support (fixed config)
      emoji_generator: !!python/name:pymdownx.emoji.to_svg
      emoji_index: !!python/name:pymdownx.emoji.twemoji

# Navigation structure (Diátaxis framework)
nav:
  - Home: index.md
  - Tutorials:
    - Getting Started: tutorials/getting-started.md
    - MkDocs Setup: tutorials/mkdocs-setup-guide.md
  - How-To:
    - Meeting Minutes: how-to/create-meeting-minutes.md
  - Reference:
    - Overview: reference/index.md
    - MkDocs Technical: reference/mkdocs-technical-reference.md
  - Explanation:
    - Overview: explanation/_index.md
    - Why DRUIDS: explanation/why-druids-matters.md

    Configuration Lessons Learned:

    • ✅ Use navigation.tabs - Solves dropdown menu issues
    • ✅ Minimal plugin set - Reduces complexity and conflicts
    • ✅ Fixed emoji config - Prevents "materialx" module errors
    • ✅ Custom CSS overlay - Revolutionary aesthetics without theme complexity
    • ❌ Avoid navigation.instant - Can break custom CSS
    • ❌ Avoid complex plugin chains - Increases debugging overhead

    Adding New Pages¶

    1. Create a new Markdown file in the docs/ directory:

    nav:

    Deployment¶

    1. GitHub Pages (Recommended)¶

    # Build and deploy to GitHub Pages
mkdocs gh-deploy

# This will:
# 1. Build the site
# 2. Push to gh-pages branch
# 3. Enable GitHub Pages automatically

    Your site will be available at: https://ks-sbc.github.io/druids-wiki/

    2. Manual Deployment¶

    # Build the site
mkdocs build

# Upload the 'site' directory to your web server
rsync -av site/ user@server:/var/www/druids-docs/

    3. Continuous Deployment¶

    Create .github/workflows/docs.yml:

    name: Deploy Documentation
on:
  push:
    branches: [ main ]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - uses: actions/setup-python@v4
      with:
        python-version: 3.x
    - run: pip install mkdocs-material
    - run: mkdocs gh-deploy --force

    DRUIDS-Specific Workflows¶

    1. Standardized Frontmatter¶

    Every DRUIDS document must include standardized YAML frontmatter:

    ---
title: "Document Title Here"
description: "One to two sentence summary of content"
type: "tutorial|how-to|reference|explanation"
security: "L0|L1|L2"
document_id: "INT-[TYPE]-2025-[SEQ]-[SECURITY]"
version: "1.0.0"
tags: [relevant, content, tags]
---

    Real Example from DRUIDS Documentation:

    ---
title: "MkDocs Setup and Usage Guide"
description: "Step-by-step tutorial for setting up and using MkDocs to build the DRUIDS documentation website"
type: "tutorial"
security: "L0"
document_id: "INT-TUT-2025-195-L0"
version: "1.0.0"
tags: ["mkdocs", "setup", "documentation", "tutorial"]
---

    Why This Matters:

    • 🔍 Search functionality - MkDocs uses title/description for search
    • 🗂️ Content organization - Type enables automated categorization
    • 🔒 Security awareness - Visual indicators based on security level
    • 📊 Analytics - Document tracking and version management

    Troubleshooting¶

    Common Issues (From Real DRUIDS Experience)¶

    1. "Command not found: mkdocs"
    # Make sure you activated the virtual environment
source mkdocs-env/bin/activate
which mkdocs  # Should show path to mkdocs

# If still not found, reinstall:
pip install mkdocs-material
    1. "Cannot find module 'pymdownx' or 'materialx'"
    # This means incomplete installation
pip install mkdocs-material  # Includes all dependencies

# Check emoji configuration in mkdocs.yml:
# WRONG: emoji_generator: !!python/name:materialx.emoji.to_svg
# RIGHT: emoji_generator: !!python/name:pymdownx.emoji.to_svg
    1. Dropdown menus not working / all opening at once
    # SOLUTION: Use Material theme navigation features
theme:
  name: material
  features:
    - navigation.tabs      # Fixes dropdown issues
    - navigation.sections  # Proper section grouping

    Lesson: Don't build custom navigation - Material theme's proven navigation works reliably.

    1. Navigation not showing new pages
    2. Check YAML indentation in mkdocs.yml (use 2 spaces, not tabs)
    3. Ensure file paths are correct and case-sensitive
    4. Restart development server: Ctrl+C then mkdocs serve

    5. Verify file exists: ls docs/path/to/file.md

    6. Custom CSS not applying

    # Ensure CSS is properly referenced
extra_css:
  - stylesheets/druids-material.css

    # Verify file exists
ls docs/stylesheets/druids-material.css

# Clear browser cache (Ctrl+Shift+R)
# Check browser dev tools for CSS errors
    1. Build fails with link warnings
    WARNING - Doc file contains a link 'missing-file.md', but target not found

    Solution: - Fix broken links to existing files - Remove aspirational links to non-existent content - Use mkdocs build (without --strict) to allow warnings

    1. Site looks wrong after switching to Material theme
    # Clear browser cache completely
# Check for conflicting custom CSS
# Verify Material theme installed: pip list | grep mkdocs-material
