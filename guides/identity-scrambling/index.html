    Identity Scrambling Techniques¶

    This is an L1 (Member-only) document. These techniques go beyond basic pseudonyms to provide comprehensive identity protection for members engaged in revolutionary organizing.

    Why Identity Scrambling Matters¶

    State and corporate surveillance capabilities grow daily. A single slip-up can connect your revolutionary activities to your legal identity, leading to:

    • Employment termination
    • State harassment or prosecution
    • Doxxing by fascist groups
    • Social/family consequences

    Identity scrambling creates multiple layers of protection, making it exponentially harder to connect your various activities.

    1. Multiple Pseudonym Strategy¶

    The Compartmentalization Principle¶

    Never use the same pseudonym across different security contexts:

    Legal Name: [NEVER USE]
     ↓
KSBC Work: "RedFox"
     ↓
Public Writing: "Prairie Voice"  
     ↓
Social Media: "Midwest Organizer"
     ↓
Other Groups: "Comrade Seven"

    Implementation¶

    • Unique per project: Different revolutionary groups = different names
    • Unique per platform: GitHub vs Signal vs Forums
    • Unique per role: Public spokesperson vs internal organizer
    • Document privately: Keep a secure (encrypted) record of which identity is used where

    2. Writing Style Variation¶

    Stylometric Fingerprinting¶

    AI can identify authors by analyzing:

    • Vocabulary choices
    • Sentence structure
    • Punctuation habits
    • Common phrases

    Techniques to Vary Style¶

    For Public Documents:

    • Use formal, academic tone
    • Longer sentences with subclauses
    • Avoid personal anecdotes
    • Heavy use of passive voice

    For Internal Documents:

    • More casual, direct tone
    • Shorter, punchy sentences
    • Include relevant examples
    • Active voice preferred

    Style Switching Checklist:

    • Change sentence length patterns
    • Rotate vocabulary (use thesaurus)
    • Vary punctuation style
    • Alter paragraph structure
    • Switch between US/UK spelling

    3. Commit Time Randomization¶

    The Problem¶

    Regular commit patterns reveal timezone and work schedule:

    Mon-Fri: commits 6-8pm → Evening worker in CST
Sat-Sun: commits 10am-2pm → Weekend schedule

    Solution: Time Scrambling¶

    Manual Method:

    # Commit with custom timestamp (±2 hours)
GIT_AUTHOR_DATE="2025-06-30 14:30:00" git commit -m "Update"

# Or use a random offset
OFFSET=$((RANDOM % 240 - 120))  # ±2 hours in minutes
NEW_TIME=$(date -d "$OFFSET minutes" '+%Y-%m-%d %H:%M:%S')
GIT_AUTHOR_DATE="$NEW_TIME" git commit -m "Update"

    Automated Script (add to .bashrc):

    git-commit-random() {
    # Random offset ±2 hours
    OFFSET=$((RANDOM % 240 - 120))
    NEW_TIME=$(date -d "$OFFSET minutes" '+%Y-%m-%d %H:%M:%S')
    GIT_AUTHOR_DATE="$NEW_TIME" GIT_COMMITTER_DATE="$NEW_TIME" git commit "$@"
}

    4. Network Anonymization¶

    VPN + Tor for Git Operations¶

    Basic Setup:

    1. VPN → Changes your IP location
    2. Tor → Adds additional anonymity layer
    3. Git over Tor → Maximum protection

    Configuration:

    # Configure Git to use Tor
git config --global http.proxy socks5://127.0.0.1:9050
git config --global https.proxy socks5://127.0.0.1:9050

# For SSH operations
# Add to ~/.ssh/config:
Host github.com
    ProxyCommand nc -x 127.0.0.1:9050 %h %p

    Rotation Strategy¶

    • Change VPN servers regularly
    • Use different VPN providers for different projects
    • Avoid patterns (same server every Tuesday)

    5. Device & Account Separation¶

    Hardware Separation (Best)¶

    • Work laptop: Legal identity only
    • KSBC laptop: Revolutionary work only
    • Public device: For spaces/cafes

    Virtual Separation (Good)¶

    • Use VMs (VirtualBox/VMware) for different identities
    • Separate user accounts on same machine
    • Different browsers per identity

    Account Separation¶

    • Separate email addresses per identity
    • Different password managers
    • No cross-identity bookmarks
    • Distinct browser profiles

    6. Metadata Scrubbing¶

    Documents¶

    # Remove metadata from PDFs
exiftool -all= document.pdf

# Remove metadata from images
exiftool -all= image.jpg

# Check what metadata exists
exiftool -a file.pdf

    Git Metadata¶

    # Check your Git identity
git config user.name
git config user.email

# Remove build paths from compiled files
strip --strip-all binary

    7. Social Media Hygiene¶

    Cross-Contamination Prevention¶

    • Never "follow" your other identities
    • Don't share content between identities
    • Avoid similar profile photos
    • Different posting schedules

    Digital Footprint Reduction¶

    • Regularly delete old posts
    • Use disappearing messages
    • Avoid location tags
    • Disable read receipts

    8. Emergency Identity Burn¶

    When to Burn an Identity¶

    • Suspected compromise
    • Accidental real name use
    • Cross-contamination detected
    • Operation completed

    Burn Procedure¶

    1. Stop all activity immediately
    2. Archive important data (encrypted)
    3. Delete accounts in reverse chronological order
    4. Wipe devices used with that identity
    5. Wait period before creating new identity
    6. Never reuse any aspect of burned identity

    Clean Wipe Commands¶

    # Secure file deletion
shred -vfz sensitive-file

# Wipe free space
# Linux:
dd if=/dev/urandom of=tempfile bs=1M; rm tempfile

# Full disk wipe (WARNING: Destroys everything)
# dd if=/dev/urandom of=/dev/sdX bs=1M

    Quick Reference Card¶

    Daily Operations¶

    • Check current pseudonym before starting work
    • Verify VPN/Tor connection active
    • Confirm correct device/VM/profile
    • Review recent commits for identity leaks

    Weekly Security Review¶

    • Rotate VPN servers
    • Check for metadata in shared files
    • Review social media for contamination
    • Update encrypted identity documentation

    Monthly Deep Clean¶

    • Full metadata audit of repositories
    • Password rotation for current identity
    • Review and update scrambling techniques
    • Practice emergency burn procedure

    Remember¶

    "The revolutionary's first duty is to survive to fight another day. Good security isn't paranoia—it's revolutionary discipline."

    Perfect security is impossible, but good security is achievable. Each layer of protection multiplies your safety. Start with the basics and gradually add more advanced techniques as you become comfortable.
