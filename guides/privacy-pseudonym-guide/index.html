    Privacy & Pseudonym Quick Start Guide¶

    • TODO: Proof read this
    • TODO: Format correctly

    Why This Matters: Revolutionary organizations face real surveillance threats. Using pseudonyms protects members from doxxing, employer retaliation, and state monitoring. This is not paranoia—it's operational security.

    1. Choose Your Pseudonym (2 minutes)¶

    Pick a code name that:

    • ✅ Is memorable but not personally identifying
    • ✅ Doesn't hint at your real identity
    • ✅ You can use consistently

    Good Examples:

    • RedFox
    • Agent7
    • Comrade191
    • BlueSparrow

    Bad Examples:

    • JohnFromKansas (location info)
    • Teacher2020 (profession hint)
    • Mike_S (partial real name)

    2. Configure Git (3 minutes)¶

    Open your terminal and run these commands:

    # Set your pseudonym as Git author
git config --global user.name "YourPseudonym"

# Use generic organizational email
git config --global user.email "member@ksbc.org"

# Verify your settings
git config --list | grep user

    Important: This affects ALL your Git repositories. For KSBC-only pseudonym:

    # Navigate to your KSBC repository first
cd /path/to/ksbc/repo

# Set config for this repository only (no --global)
git config user.name "YourPseudonym"
git config user.email "member@ksbc.org"

    3. Common Mistakes to Avoid¶

    ❌ Don't Use Your GitHub Username¶

    Your GitHub profile might have real information. Use a different pseudonym for commits.

    ❌ Don't Reference Personal Details¶

    Avoid pseudonyms like:

    • Birth years (Agent1995)
    • Locations (KansasComrade)
    • Interests that could identify you (GuitarMarxist)

    ❌ Don't Forget Other Repositories¶

    If you contribute to multiple projects, decide whether to:

    • Use the same pseudonym everywhere (simpler)
    • Use different pseudonyms per project (more secure)

    ❌ Don't Put Real Names in Code Comments¶

    Check your code for:

    # Bad: Created by John Smith
# Good: Created for KSBC project

    4. Quick Security Check¶

    Run this command to see what name appears in your recent commits:

    git log --format="%an <%ae>" -10

    If you see your real name, you'll need to either:

    1. Start fresh with new commits using your pseudonym
    2. Clean your Git history (see Git History Scrubbing Guide)

    5. Beyond Git: Document Privacy¶

    When writing documentation:

    • Never include member real names
    • Use pseudonyms or roles ("the secretary", "a member")
    • Redact identifying information from screenshots
    • Check meeting minutes for accidental name drops

    6. Emergency: I Accidentally Used My Real Name¶

    Don't panic. Take these steps:

    1. Stop pushing commits immediately
    2. Check if it's pushed: git status
    3. If not pushed: Amend your last commit:
    git commit --amend --author="YourPseudonym <member@ksbc.org>"
    1. If already pushed: Contact security team for Git history cleaning

    Next Steps¶

    • Configure your Git settings now
    • Check your recent commits for real names
    • Review any documents you've written
    • Read the Identity Scrambling Guide for advanced protection

    Quick Reference Card¶

    # Initial setup
git config user.name "YourPseudonym"
git config user.email "member@ksbc.org"

# Check current config
git config user.name
git config user.email

# Check recent commits
git log --format="%an <%ae>" -10

# Fix last commit (if not pushed)
git commit --amend --author="YourPseudonym <member@ksbc.org>"

    Remember: Good security is a practice, not a one-time setup. Stay vigilant!
