    How to Create the MkDocs Mandalorian Theme¶

    • TODO: Proof read this
    • TODO: Format correctly

    Overview¶

    This guide walks you through creating a complete MkDocs theme package implementing the Mandalorian aesthetic for technical documentation.

    Prerequisites¶

    • Python 3.7+
    • MkDocs 1.0+
    • Basic understanding of Jinja2 templates
    • Familiarity with the Mandalorian aesthetic specs

    Basic Setup¶

    1. Create Theme Directory Structure¶

    mkdir -p mandalorian-theme/mandalorian
cd mandalorian-theme

# Create required files
touch setup.py
touch MANIFEST.in
touch README.md
touch mandalorian/__init__.py
touch mandalorian/mkdocs_theme.yml

# Create template directory
mkdir -p mandalorian/{css,js,img}
touch mandalorian/main.html
touch mandalorian/base.html
touch mandalorian/nav.html

    2. Configure Theme Package¶

    setup.py¶

    from setuptools import setup, find_packages

setup(
    name='mkdocs-mandalorian-theme',
    version='1.0.0',
    description='Mandalorian theme for DRUIDS technical documentation',
    author='KSBC DRUIDS Team',
    author_email='tech@ksbc.org',
    url='https://github.com/ksbc/mandalorian-theme',
    packages=find_packages(),
    include_package_data=True,
    entry_points={
        'mkdocs.themes': [
            'mandalorian = mandalorian',
        ]
    },
    zip_safe=False,
    python_requires='>=3.7',
    install_requires=['mkdocs>=1.0'],
    classifiers=[
        'Development Status :: 5 - Production/Stable',
        'Programming Language :: Python :: 3',
        'License :: OSI Approved :: MIT License',
    ],
)

    MANIFEST.in¶

    recursive-include mandalorian *.html *.css *.js *.png *.svg *.yml
include README.md
include LICENSE

    mandalorian/mkdocs_theme.yml¶

    # Theme configuration
theme_name: Mandalorian
theme_version: 1.0.0

# Extends base theme (optional)
extends: null

# Static files
static_templates:
  - 404.html

# Theme options
vars:
  # Colors
  primary_color: '#4a8b4a'  # Beskar green
  accent_color: '#c76e2e'   # Rust orange
  background_color: '#1a1d23'  # Dark slate
  text_color: '#e0e0e0'     # Off-white

  # Typography
  heading_font: 'Bebas Neue'
  body_font: 'Inter'
  code_font: 'JetBrains Mono'

  # Features
  include_search: true
  include_toc: true
  security_indicators: true

    Template Implementation¶

    1. Base Template (base.html)¶

    <!DOCTYPE html>
<html lang="{{ config.theme.language }}">
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    {% if page and page.meta and page.meta.title %}
        <title>{{ page.meta.title }} - {{ config.site_name }}</title>
    {% elif page and page.title %}
        <title>{{ page.title }} - {{ config.site_name }}</title>
    {% elif config.site_name %}
        <title>{{ config.site_name }}</title>
    {% endif %}

    <!-- Favicon -->
    <link rel="icon" href="{{ 'img/favicon.ico'|url }}">

    <!-- Fonts -->
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link href="https://fonts.googleapis.com/css2?family=Bebas+Neue&family=Inter:wght@400;500;600&family=JetBrains+Mono&display=swap" rel="stylesheet">

    <link rel="stylesheet" href="{{ 'css/security.css'|url }}">

    <!-- Custom CSS -->
    {% for path in config.extra_css %}
        <link rel="stylesheet" href="{{ path|url }}">
    {% endfor %}

    {% block extrahead %}{% endblock %}
</head>
<body class="mandalorian-theme">
    <!-- Scratched metal texture overlay -->
    <div class="metal-texture"></div>

    <div class="wrapper">
        <!-- Navigation -->
        {% include "nav.html" %}

        <!-- Main content -->
        <main class="main-content">
            {% block content %}{% endblock %}
        </main>

        <!-- Footer -->
        <footer class="site-footer">
            <div class="footer-content">
                {% if config.copyright %}
                    <p>{{ config.copyright }}</p>
                {% endif %}
                <p>Built with <a href="https://www.mkdocs.org">MkDocs</a> using Mandalorian theme</p>
            </div>
        </footer>
    </div>

    <!-- Theme JavaScript -->
    <script src="{{ 'js/theme.js'|url }}"></script>

    <!-- Custom JavaScript -->
    {% for path in config.extra_javascript %}
        <script src="{{ path|url }}"></script>
    {% endfor %}
</body>
</html>

    2. Main Content Template (main.html)¶

    {% extends "base.html" %}

{% block content %}
<div class="container">
    <!-- Security indicator based on page metadata -->
    {% if page and page.meta and page.meta.security %}
        <div class="security-indicator security-{{ page.meta.security|lower }}">
            <span class="security-badge">{{ page.meta.security }}</span>
        </div>
    {% endif %}

    <!-- Breadcrumbs -->
    <nav class="breadcrumb" aria-label="breadcrumb">
        <ol>
            <li><a href="{{ config.site_url }}">Home</a></li>
            {% if page %}
                {% for doc in page.ancestors %}
                    <li><a href="{{ doc.url|url }}">{{ doc.title }}</a></li>
                {% endfor %}
                <li class="active">{{ page.title }}</li>
            {% endif %}
        </ol>
    </nav>

    <!-- Page header -->
    <header class="page-header">
        <h1>{{ page.title }}</h1>
        {% if page and page.meta %}
            <div class="page-meta">
                {% if page.meta.date %}
                    <time datetime="{{ page.meta.date }}">{{ page.meta.date }}</time>
                {% endif %}
                {% if page.meta.authors %}
                    <span class="authors">By {{ page.meta.authors|join(', ') }}</span>
                {% endif %}
            </div>
        {% endif %}
    </header>

    <!-- Table of contents -->
    {% if page.toc %}
        <nav class="toc" aria-label="Table of contents">
            <h2>Contents</h2>
            <ul>
                {% for item in page.toc %}
                    <li>
                        <a href="{{ item.url }}">{{ item.title }}</a>
                        {% if item.children %}
                            <ul>
                                {% for child in item.children %}
                                    <li><a href="{{ child.url }}">{{ child.title }}</a></li>
                                {% endfor %}
                            </ul>
                        {% endif %}
                    </li>
                {% endfor %}
            </ul>
        </nav>
    {% endif %}

    <!-- Page content -->
    <article class="page-content">
        {{ page.content }}
    </article>

    <!-- Page navigation -->
    <nav class="page-nav">
        {% if page.previous_page %}
            <a href="{{ page.previous_page.url|url }}" class="nav-prev">
                ← {{ page.previous_page.title }}
            </a>
        {% endif %}
        {% if page.next_page %}
            <a href="{{ page.next_page.url|url }}" class="nav-next">
                {{ page.next_page.title }} →
            </a>
        {% endif %}
    </nav>
</div>
{% endblock %}

    JavaScript Enhancement¶

    Theme JavaScript (js/theme.js)¶

    // Mandalorian Theme JavaScript

(function() {
  'use strict';

  // Security level detection
  function detectSecurityLevel() {
    const path = window.location.pathname;
    if (path.includes('/public/')) return 'l0';
    if (path.includes('/member/')) return 'l1';
    if (path.includes('/cadre/')) return 'l2';
    return 'l0'; // default
  }

  // Apply security class to body
  document.addEventListener('DOMContentLoaded', function() {
    const securityLevel = detectSecurityLevel();
    document.body.classList.add(`security-${securityLevel}`);

    // Add visual indicator if not already present
    const mainContent = document.querySelector('.page-content');
    if (mainContent && !document.querySelector('.security-indicator')) {
      const indicator = document.createElement('div');
      indicator.className = `security-indicator security-${securityLevel}`;
      indicator.innerHTML = `<span class="security-badge">${securityLevel.toUpperCase()}</span>`;
      mainContent.parentNode.insertBefore(indicator, mainContent);
    }
  });

  // Smooth scroll for anchor links
  document.querySelectorAll('a[href^="#"]').forEach(anchor => {
    anchor.addEventListener('click', function(e) {
      e.preventDefault();
      const target = document.querySelector(this.getAttribute('href'));
      if (target) {
        target.scrollIntoView({ behavior: 'smooth' });
      }
    });
  });

  // Code block enhancements
  document.querySelectorAll('pre code').forEach(block => {
    // Add line numbers
    const lines = block.textContent.split('\n');
    if (lines.length > 1) {
      block.classList.add('line-numbers');
    }

    // Add copy button
    const button = document.createElement('button');
    button.className = 'copy-code';
    button.textContent = 'Copy';
    button.addEventListener('click', function() {
      navigator.clipboard.writeText(block.textContent);
      button.textContent = 'Copied!';
      setTimeout(() => button.textContent = 'Copy', 2000);
    });
    block.parentNode.appendChild(button);
  });
})();

    Installation & Usage¶

    1. Install the Theme¶

    # Development installation
cd mandalorian-theme
pip install -e .

# Production installation
pip install mkdocs-mandalorian-theme

    2. Configure MkDocs¶

    In mkdocs.yml:

    site_name: DRUIDS Technical Documentation
site_url: https://docs.druids.ksbc.org
site_description: Technical documentation for DRUIDS infrastructure

theme:
  name: mandalorian
  palette:
    primary: beskar
    accent: rust
  features:
    - navigation.instant
    - navigation.tabs
    - toc.autohide

  # Custom variables
  custom:
    security_indicators: true
    show_breadcrumbs: true

# Extensions
markdown_extensions:
  - admonition
  - codehilite:
      guess_lang: false
  - toc:
      permalink: true
  - pymdownx.superfences
  - pymdownx.tabbed

# Security metadata example
nav:
  - Home: index.md
  - Public Docs:
    - Getting Started: public/getting-started.md
  - Member Docs:
    - Development: member/development.md
  - Technical Specs:
    - Architecture: technical/architecture.md

    Troubleshooting¶

    Theme Not Loading¶

    1. Verify installation: pip show mkdocs-mandalorian-theme
    2. Check theme name in mkdocs.yml
    3. Clear MkDocs cache: mkdocs build --clean

    Styling Issues¶

    1. Check browser console for CSS errors
    2. Verify font loading
    3. Test in different browsers

    Security Indicators Not Showing¶

    1. Ensure metadata is properly formatted
    2. Check JavaScript console
    3. Verify CSS is loaded
