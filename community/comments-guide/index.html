    Comments and Discussion Guide¬∂

    The DRUIDS wiki uses Giscus for comments, leveraging GitHub Discussions for a privacy-respecting, community-driven discussion system.

    Overview¬∂

    Giscus provides:

    • Privacy-Respecting: No tracking, ads, or cookies beyond GitHub authentication
    • GitHub Integration: Uses GitHub Discussions as the backend
    • Mandalorian Theme: Custom styling matching our tactical aesthetic
    • Security: Origin restrictions and GitHub's authentication

    How to Enable Comments¬∂

    To enable comments on any documentation page, add the following to the page's frontmatter:

    ---
comments: true
---

    Comments are disabled by default on all pages for security and focus.

    Using the Comment System¬∂

    Posting Comments¬∂

    1. Authentication: Click "Sign in with GitHub" to authenticate
    2. Write: Use the comment box with Markdown support
    3. Preview: Check your comment before posting
    4. Submit: Post your comment to the discussion

    • Editing: Edit your own comments after posting

    Moderation Guidelines¬∂

    Community Standards¬∂

    1. Be Respectful: Treat all members with respect
    2. Stay On Topic: Keep discussions relevant to the documentation
    3. Constructive Feedback: Provide helpful, actionable feedback
    4. No Spam: Avoid promotional or off-topic content

    Moderation Tools¬∂

    Repository maintainers can:

    • Lock discussions to prevent new comments
    • Hide inappropriate comments
    • Delete spam or offensive content
    • Transfer discussions to more appropriate categories

    Technical Implementation¬∂

    Architecture¬∂

    graph LR
    A[DRUIDS Wiki] -->|embeds| B[Giscus Widget]
    B -->|authenticates| C[GitHub OAuth]
    B -->|stores in| D[GitHub Discussions]
    D -->|returns to| B
    B -->|displays in| A

    Security Features¬∂

    1. Origin Restrictions: Only allowed domains can embed comments
    {
  "origins": [
    "https://druids.kssocialistbookclub.com",
    "http://localhost:8000"
  ]
}
    1. Authentication: GitHub's OAuth ensures users are authenticated
    2. Content Security: GitHub's anti-spam and content policies apply

    Theme Integration¬∂

    Comments automatically adapt to the site's theme:

    • Dark Mode: Mandalorian tactical theme
    • Light Mode: Currently uses the same theme (customizable)
    • Custom CSS: Located at /assets/stylesheets/giscus-druids.css

    Best Practices¬∂

    For Contributors¬∂

    1. Search First: Check if your question has been answered
    2. Be Specific: Provide context and examples
    3. Use Code Blocks: Format code properly for readability
    4. Link Issues: Reference related GitHub issues when relevant

    For Maintainers¬∂

    1. Regular Review: Check discussions weekly
    2. Quick Response: Acknowledge questions within 48 hours
    3. Documentation Updates: Convert FAQs into documentation
    4. Community Building: Encourage helpful contributors

    Troubleshooting¬∂

    Comments Not Appearing¬∂

    1. Check Frontmatter: Ensure comments: true is set
    2. GitHub Login: Verify you're logged into GitHub
    3. Browser Extensions: Disable ad blockers temporarily
    4. Clear Cache: Try hard refresh (Ctrl+Shift+R)

    Styling Issues¬∂

    1. Theme Conflicts: Check browser console for CSS errors
    2. Custom Themes: Ensure Giscus CSS is loaded after Material theme
    3. Mobile View: Comments are responsive but may stack vertically

    Repository Configuration¬∂

    Ensure your GitHub repository has:

    1. Discussions Enabled: Settings ‚Üí Features ‚Üí Discussions
    2. Public Access: Comments require public repository
    3. Giscus App: Install from giscus.app

    Advanced Configuration¬∂

    Custom Categories¬∂

    Create discussion categories in GitHub for:

    • Documentation Feedback
    • Feature Requests
    • Bug Reports
    • General Discussion

    Webhook Integration¬∂

    For advanced workflows, configure GitHub webhooks to:

    • Notify maintainers of new comments
    • Sync comments to external systems
    • Generate comment analytics

    Privacy Considerations¬∂

    Data Storage¬∂

    • All comments stored in GitHub's infrastructure
    • No additional tracking beyond GitHub's analytics
    • Users control their own data through GitHub

    GDPR Compliance¬∂

    • No cookies set by Giscus
    • GitHub handles data processing agreements
    • Users can delete their own comments

    Future Enhancements¬∂

    Planned improvements:

    1. L1/L2 Comment Sections: Separate discussions by security level
    2. Auto-Translation: Multi-language comment support
    3. AI Moderation: Automated spam detection
    4. Knowledge Base Integration: Convert discussions to documentation

    For technical support with comments, please open an issue on GitHub.
