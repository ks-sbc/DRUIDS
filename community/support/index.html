    Get Support¶

    Overview¶

    DRUIDS is designed for self-sufficiency, but we understand that questions arise during implementation. This guide covers all support options, from self-help resources to community assistance.

    Best for: Feature requests, implementation strategies, sharing experiences

    • 💡 Ideas & Features

    1. Search existing discussions
    2. Check documentation

    4. Include your setup details

    Issue Reporting¶

    For bugs and documentation issues only

    URL: github.com/your-org/druids/issues

    Good Issue Template:

    ## Description
Clear description of the issue

## Steps to Reproduce
1. Step one
2. Step two
3. Error occurs

## Expected Behavior
What should happen

## Actual Behavior
What actually happens

## Environment
- OS: [e.g., Ubuntu 22.04]
- Obsidian: [version]
- DRUIDS: [version]
- Relevant plugins: [versions]

## Additional Context
Any other relevant information

    Security Issues¶

    DO NOT post security vulnerabilities publicly!

    Contact: security@your-org.example Response Time: Within 48 hours

    Include:

    • Vulnerability description
    • Impact assessment
    • Steps to reproduce (privately)
    • Your contact information

    Common Issues & Solutions¶

    Installation Problems¶

    "Git not found" Error¶

    # Ubuntu/Debian
sudo apt-get install git

# macOS
brew install git

# Windows
winget install --id Git.Git

    Obsidian-Git Not Syncing¶

    1. Check authentication: git push origin main
    2. Verify remote: git remote -v
    3. Test SSH: ssh -T git@github.com
    4. Check credentials: Git credential manager

    Security Classification Issues¶

    Visual Indicators Not Showing¶

    1. Verify CSS is loaded
    2. Check snippet is enabled in settings
    3. Confirm frontmatter format
    4. Restart Obsidian

    Wrong Repository Errors¶

    1. Check current branch: git branch
    2. Verify security classification
    3. Ensure in correct vault

    Performance Issues¶

    Slow Vault Loading¶

    1. Check vault size: du -sh vault/
    2. Disable unused plugins
    3. Optimize attachments
    4. Consider vault splitting

    Git Operations Timing Out¶

    1. Reduce commit frequency
    2. Check internet connection
    3. Optimize .gitignore
    4. Consider Git LFS for large files

    Support Etiquette¶

    DO's ✅¶

    • Search before asking
    • Provide context and examples
    • Be patient and respectful
    • Follow up with solutions
    • Thank helpers

    DON'Ts ❌¶

    • Post security issues publicly
    • Demand immediate responses
    • Cross-post same question
    • Share sensitive organization data
    • Assume malice
