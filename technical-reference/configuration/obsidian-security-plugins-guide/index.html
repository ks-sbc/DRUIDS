    DRUIDS Security Plugins Configuration Guide¶

    Overview¶

    This guide covers the security-focused plugin configurations that protect DRUIDS infrastructure and enforce organizational security policies.

    Plugin Stack¶

    1. GPGCrypt - Encryption Layer¶

    Purpose: Encrypts sensitive L2 content with GPG

    Configuration:

    {
  "keyId": "",
  "method": "gpg",
  "binary": "/usr/bin/gpg",
  "armor": true,
  "alwaysEncrypt": false,
  "alwaysDecrypt": true,
  "trustModel": "always",
  "defaultRecipient": "",
  "encryptionAlgorithm": "AES256",
  "smartcardSupport": true,
  "alwaysSign": true,
  "requireStrongKey": true,
  "minKeyLength": 4096
}

    Key Features:

    • 4096-bit minimum key length
    • AES256 encryption standard
    • Smartcard support for key storage
    • Automatic signing of encrypted content

    2. Linter - Document Standards Enforcement¶

    Purpose: Validates DRUIDS document requirements

    Custom Rules:

    "customRules": [
  {
    "name": "DRUIDS Document ID Required",
    "regex": "^---[\\s\\S]*?document_id:\\s*[\\w-]+[\\s\\S]*?---",
    "replacement": "",
    "error": "Missing required 'document_id' field in frontmatter"
  },
  {
    "name": "Security Level Required",
    "regex": "^---[\\s\\S]*?security_level:\\s*(L0|L1|L2|public|member|cadre)[\\s\\S]*?---",
    "replacement": "",
    "error": "Missing or invalid 'security_level' field"
  },
  {
    "name": "Version Format Check",
    "regex": "^---[\\s\\S]*?version:\\s*\\d+\\.\\d+\\.\\d+[\\s\\S]*?---",
    "replacement": "",
    "error": "Version must follow semantic versioning (X.Y.Z)"
  }
]

    3. Style Settings - Visual Security Indicators¶

    Purpose: Provides immediate visual security awareness

    Configuration:

    {
  "druids-security-theme": {
    "l0-color": "#22c55e",
    "l1-color": "#f59e0b",
    "l2-color": "#ef4444",
    "show-file-badges": true,
    "show-tab-indicators": true,
    "show-graph-colors": true
  }
}

    4. Shell Commands - Security Automation¶

    Purpose: Automated security operations

    Security Commands:

    [
  {
    "name": "Security Audit",
    "command": "/repos/shared/scripts/bash/security-audit.sh {{vault_path}}",
    "output": "modal"
  },
  {
    "name": "Check Classification",
    "command": "find . -name '*.md' -exec grep -l 'security_level' {} \\; | xargs -I {} bash -c 'echo -n \"{}: \"; grep security_level {} | head -1'",
    "output": "notification"
  }
]

    Security Workflows¶

    Document Classification Workflow¶

    1. Creation: Template auto-detects security from path
    2. Validation: Linter checks frontmatter compliance
    3. Visual: Style Settings shows security colors
    4. Audit: Shell command verifies placement

    Encryption Workflow (L2 Only)¶

    1. Write: Create content in .md file
    2. Encrypt: Use GPGCrypt to encrypt
    3. Save: Store as .md.gpg
    4. Version Control: Commit encrypted file only

    Security Review Process¶

    graph TD
    A[New Document] --> B{Security Check}
    B -->|L0| C[Public Vault]
    B -->|L1| D[Member Vault]
    B -->|L2| E[Cadre Vault + Encrypt]
    C --> F[Visual Indicators]
    D --> F
    E --> F
    F --> G[Git Commit]

    Configuration Best Practices¶

    GPGCrypt Setup¶

    1. Key Generation:
    gpg --full-generate-key
# Select: RSA and RSA
# Keysize: 4096
# Expiry: 2 years
    1. Smartcard Configuration:
    gpg --card-edit
# admin
# generate
    1. Trust Model:
    2. Use web of trust for L1
    3. Require direct signatures for L2

    Linter Configuration¶

    1. Rule Priority:
    2. Security rules: High
    3. Format rules: Medium

    4. Style rules: Low

    5. Folder Exclusions:

    "folderExclusions": [
  "templates",
  ".trash",
  "archive"
]
    1. Performance:
    2. Run on save only
    3. Exclude large files
    4. Cache validation results

    Visual Security Setup¶

    1. CSS Snippet:
    /* Flashing warning for misplaced documents */
.nav-file-title[data-path*="/public/"][data-security="L2"] {
  animation: security-alert 1s infinite;
}

@keyframes security-alert {
  0%, 100% { background-color: #ef4444; }
  50% { background-color: transparent; }
}
    1. Graph View:
    2. L0: Green nodes
    3. L1: Yellow nodes
    4. L2: Red nodes

    Shell Command Security¶

    1. Command Validation:
    # Whitelist commands
ALLOWED_COMMANDS="ls find grep security-audit"
    1. Input Sanitization:
    const sanitized = input.replace(/[;&|`$]/g, '');
    1. Output Handling:
    2. Never expose full paths
    3. Limit output size
    4. Sanitize error messages

    Security Policies¶

    Access Control¶

    Vault Encryption Access Backup
    L0 Public None All members Public Git
    L1 Member Optional Members only Private Git
    L2 Cadre Mandatory Cadre only Encrypted + Offline

    Validation Rules¶

    1. Document Placement:
    2. Security level must match vault
    3. No L1/L2 content in public

    4. No unencrypted L2 content

    5. Metadata Requirements:

    6. All fields from template
    7. Valid security classification

    8. Proper version format

    9. Encryption Standards:

    10. 4096-bit RSA minimum
    11. AES256 for symmetric
    12. Sign all encrypted content

    Monitoring and Audit¶

    Daily Checks¶

    #!/bin/bash
# Daily security audit
echo "=== DRUIDS Security Audit ==="
echo "Date: $(date)"

# Check for unencrypted L2 files
echo "Checking L2 vault encryption..."
find /vault/cadre -name "*.md" ! -name "*.gpg" -type f

# Verify security classifications
echo "Checking security classifications..."
/repos/shared/scripts/bash/check-classifications.sh

# Review access logs
echo "Recent access attempts..."
grep "DENIED" /var/log/druids-access.log | tail -20

    Weekly Reports¶

    Generate security metrics:

    • Classification compliance rate
    • Encryption coverage
    • Failed access attempts
    • Policy violations

    Incident Response¶

    1. Detection: Monitor logs and alerts
    2. Containment: Isolate affected vault
    3. Investigation: Audit trail review
    4. Remediation: Fix and patch
    5. Documentation: Update procedures

    Troubleshooting¶

    GPGCrypt Issues¶

    Problem: Cannot decrypt files

    • Check key availability: gpg --list-keys
    • Verify key trust: gpg --edit-key KEY_ID
    • Test manually: gpg -d file.gpg

    Problem: Smartcard not detected

    • Install drivers: apt install pcscd
    • Check card: gpg --card-status
    • Reset card if needed

    Linter False Positives¶

    Problem: Valid documents flagged

    • Review regex patterns
    • Check for edge cases
    • Add exceptions carefully

    Visual Indicators Missing¶

    Problem: Colors not showing

    • Enable CSS snippet
    • Check theme compatibility
    • Clear CSS cache
    • Verify selector accuracy

    Advanced Security Features¶

    Multi-Factor Authentication¶

    1. Smartcard + PIN: For L2 access
    2. TOTP: For vault unlock
    3. Biometric: Optional addition

    Audit Logging¶

    // Log all security operations
const logSecurityEvent = (event) => {
  const entry = {
    timestamp: new Date(),
    user: process.env.USER,
    event: event,
    vault: getCurrentVault()
  };
  appendToLog('/var/log/druids-security.log', entry);
};

    # Auto-quarantine suspicious files
if [[ "$SECURITY_CHECK" == "FAIL" ]]; then
  mv "$FILE" "/quarantine/"

fi
