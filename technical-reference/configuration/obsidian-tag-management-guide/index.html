    Obsidian Tag Management Plugins Guide¶

    Overview¶

    This guide covers the tag management plugins (Tag Wrangler and TagFolder) configured for DRUIDS organizational taxonomy and navigation.

    Tag Wrangler Configuration¶

    Purpose¶

    Tag Wrangler provides comprehensive tag management including aliases, validation, auto-tagging, and bulk operations.

    Tag Hierarchy Structure¶

    security/
├── L0 (Public)
├── L1 (Member)
└── L2 (Cadre)

priority/
├── p0 (Critical)
├── p1 (High)
├── p2 (Medium)
├── p3 (Low)
├── p4 (Very Low)
└── p5 (Minimal)

status/
├── draft
├── review
├── voting
├── approved
└── archived

committee/
├── general
├── membership
├── education
├── security
└── finance

type/
├── meeting
├── proposal
├── report
├── task
├── decision
├── csc
└── audit

    Tag Aliases¶

    Quick shortcuts for common tags:

    Alias Expands To
    sec security
    mtg meeting
    prop proposal
    impl implementation
    csc criticism-self-criticism
    L0 security/L0
    L1 security/L1
    L2 security/L2

    Auto-Tagging Rules¶

    Tags are automatically applied based on document metadata:

    1. Document Type Detection
    2. If document_id contains -mtg- → adds #meeting

    3. If document_id contains -prop- → adds #proposal

    4. Security Level Sync

    5. If security_level: L0 → adds #security/L0
    6. If security_level: L1 → adds #security/L1
    7. If security_level: L2 → adds #security/L2

    Tag Validation Rules¶

    Security Tags¶

    • Pattern: ^security/(L0|L1|L2)$
    • Valid: #security/L0, #security/L1, #security/L2
    • Invalid: #L0, #security/L3, #security

    Priority Tags¶

    • Pattern: ^priority/p[0-5]$
    • Valid: #priority/p0 through #priority/p5
    • Invalid: #p0, #priority/p6, #priority/high

    Committee Tags¶

    • Pattern: ^committee/(general|membership|education|security|finance)$
    • Valid: Listed committees only
    • Invalid: #committee/other, #membership

    Tag Rename Rules¶

    Automatic corrections applied:

    1. Security Format
    2. #L0 → #security/L0
    3. #L1 → #security/L1

    4. #L2 → #security/L2

    5. Priority Format

    6. #p0 → #priority/p0

    7. #p1 → #priority/p1

    8. Committee Typos

    9. #comm/general → #committee/general

    Blacklisted Tags¶

    These tags are automatically removed:

    • #temp
    • #test
    • #delete-me
    • #todo

    Tag Statistics¶

    • Shows top 10 tags in status bar
    • Tracks tag usage across vaults
    • Identifies orphaned tags

    TagFolder Configuration¶

    Purpose¶

    TagFolder creates a virtual folder structure based on tags, enabling navigation by taxonomy rather than file location.

    Virtual Folder Structure¶

    Pre-configured virtual folders:

    1. 🔒 Security Classifications
    2. Shows all documents by security level

    3. Sorted: L2 → L1 → L0 (highest to lowest)

    4. 🚨 High Priority

    5. Filters for #priority/p0 and #priority/p1

    6. Critical items requiring immediate attention

    7. 🗳️ Active Votes

    8. Shows all #status/voting documents

    9. Current democratic decisions

    10. ⏸️ Blocked Items

    11. Displays #status/blocked items

    12. Identifies workflow bottlenecks

    13. 📅 This Quarter

    14. Time-based organization
    15. Shows current quarter tags

    Display Settings¶

    • Show Path: Yes (see full file location)
    • Expand Limit: 2 levels (prevents overwhelming display)
    • File Count: Enabled (shows number per tag)
    • Pinned Tags: Quick access to critical tags

    Color Coding¶

    Consistent with security theme:

    Tag Category Color
    Security Red base, varies by level
    Priority Orange spectrum
    Status Purple spectrum
    Committee Green
    Type Blue

    Custom Sort Orders¶

    1. Security: L2 → L1 → L0 (highest first)
    2. Priority: p0 → p1 → p2 → p3 → p4 → p5
    3. Status: blocked → draft → review → voting → approved → archived

    Performance Settings¶

    • Lazy Loading: Enabled for large vaults
    • Cache Timeout: 5 minutes
    • Max Items: 100 per folder
    • Virtual Scrolling: Enabled

    Integration Between Plugins¶

    Tag Wrangler → TagFolder Flow¶

    1. Tag Wrangler validates and corrects tags
    2. Auto-tagging applies consistent taxonomy
    3. TagFolder organizes by validated tags
    4. Virtual folders reflect clean hierarchy

    With Other Plugins¶

    Tasks Plugin

    • Tags determine task categories
    • Status tags sync with task states

    Kanban Plugin

    • Tag colors maintained in cards
    • Filter boards by tags

    Dataview Plugin

    • Query by tag hierarchies
    • Use validated tag names

    Best Practices¶

    Tag Creation¶

    1. Always use hierarchical format
    2. Follow validation patterns
    3. Use aliases for speed
    4. Let auto-tagging work

    Tag Management¶

    1. Regular cleanup with Tag Wrangler
    2. Review tag statistics monthly
    3. Update aliases as needed
    4. Monitor blacklist effectiveness

    Navigation¶

    1. Pin frequently used tags
    2. Use virtual folders for workflows
    3. Combine with search for precision
    4. Leverage color coding

    Common Workflows¶

    Adding New Committee¶

    1. Update Tag Wrangler validation rule
    2. Add to committee tag group
    3. Create virtual folder if needed
    4. Update documentation

    Bulk Tag Rename¶

    1. Use Tag Wrangler rename function
    2. Select old tag
    3. Enter new hierarchical tag
    4. Confirm changes
    5. Check TagFolder updates

    Security Audit by Tags¶

    1. Open Security Classifications folder
    2. Review document distribution
    3. Check for missing security tags
    4. Identify misclassified items

    Troubleshooting¶

    Tags Not Validating¶

    • Check exact pattern match
    • Verify hierarchy delimiter (/)
    • Look for typos
    • Review validation rules

    Auto-Tags Not Applying¶

    • Confirm trigger conditions
    • Check frontmatter format
    • Verify rule configuration
    • Test with sample document

    TagFolder Not Updating¶

    • Clear cache (5-minute timeout)
    • Reload Obsidian
    • Check tag existence
    • Verify no exclusions

    Performance Issues¶

    • Reduce max items per folder
    • Enable lazy loading
    • Limit expand depth
    • Archive old tags

    Advanced Configuration¶

    Custom Tag Groups¶

    {
  "name": "Project Phases",
  "tags": ["phase/planning", "phase/implementation", "phase/review"],
  "color": "#10b981"
}

    Complex Validation Rules¶

    {
  "name": "Date-based tags",
  "pattern": "^\\d{4}-(Q[1-4]|\\d{2})$",
  "appliesTo": ["temporal"]
}

    Batch Operations¶

    // Rename all old format tags
const oldTags = ["L0", "L1", "L2"];
oldTags.forEach(tag => {
  wrangler.rename(tag, `security/${tag}`);
});

    • /repos/shared/obsidian/plugins/tag-wrangler/data.json
    • /repos/shared/obsidian/plugins/tagfolder/data.json
