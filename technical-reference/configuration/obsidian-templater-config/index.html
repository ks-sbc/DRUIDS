    Obsidian Templater Plugin Configuration¶

    Overview¶

    Templater provides advanced templating capabilities with dynamic content generation, system command integration, and folder-based automation for DRUIDS document workflows.

    Core Configuration¶

    Template Settings¶

    {
  "template_folder": "templates",
  "trigger_on_file_creation": true,
  "auto_jump_to_cursor": true,
  "enable_system_commands": true,
  "shell_path": "/bin/bash"
}

    Key Features:

    • Templates stored in /templates folder
    • Automatic template application on file creation
    • Cursor positioning support
    • System command execution enabled

    Folder Template Mappings¶

    Automatic Template Assignment¶

    "folder_templates": [
  {
    "folder": "meetings",
    "template": "templates/meeting-template.md"
  },
  {
    "folder": "proposals", 
    "template": "templates/proposal-template.md"
  },
  {
    "folder": "reports",
    "template": "templates/report-template.md"
  },
  {
    "folder": "csc",
    "template": "templates/csc-template.md"
  },
  {
    "folder": "tasks",
    "template": "templates/task-template.md"
  }
]

    Usage: Create a file in any mapped folder to automatically apply the corresponding template.

    System Commands Configuration¶

    Document ID Generation¶

    {
  "name": "GenerateDocumentID",
  "command": "node /repos/shared/scripts/js/generate-document-id.js"
}

    Usage in Template:

    ---
document_id: <% tp.system.command("GenerateDocumentID") %>
---

    Security Level Detection¶

    {
  "name": "GetSecurityLevel",
  "command": "pwd | grep -o 'L[0-2]\\|public\\|member\\|cadre' | head -1"
}

    Automatically detects security level from vault path.

    User Role Detection¶

    {
  "name": "GetUserRole",
  "command": "echo ${DRUIDS_USER_ROLE:-member}"
}

    Reads from environment variable with fallback.

    Template Functions¶

    Date Formatting¶

    <% tp.date.now("YYYY-MM-DD") %>           // Current date
<% tp.date.now("YYYY-MM-DD HH:mm") %>     // Date and time
<% tp.date.now("YYYY-MM") %>              // Year-month
<% tp.date.weekday("YYYY-MM-DD", 1) %>    // Next Monday

    File Operations¶

    <% tp.file.title %>                        // Current file name
<% tp.file.folder() %>                     // Current folder
<% tp.file.path(true) %>                   // Relative path
<% tp.file.creation_date("YYYY-MM-DD") %> // File creation date

    Custom Functions¶

    <% tp.user.getDocumentID() %>              // Generate document ID
<% tp.user.getNextSequence() %>            // Get sequence number
<% tp.user.getSecurityFromPath() %>        // Detect security level

    Example Script¶

    // get-next-meeting-date.js
function getNextMeetingDate(dayOfWeek) {
  const days = ['sunday','monday','tuesday','wednesday','thursday','friday','saturday'];
  const today = new Date();
  const targetDay = days.indexOf(dayOfWeek.toLowerCase());

  let daysUntilTarget = targetDay - today.getDay();
  if (daysUntilTarget <= 0) daysUntilTarget += 7;

  const nextDate = new Date(today);
  nextDate.setDate(today.getDate() + daysUntilTarget);

  return nextDate.toISOString().split('T')[0];
}

module.exports = getNextMeetingDate;

    Security Considerations¶

    System Commands¶

    • Only whitelisted commands allowed
    • Commands run with user permissions
    • No network access in templates
    • Input validation required

    Template Execution¶

    • Templates run in sandboxed environment
    • File system access limited to vault
    • No external script execution
    • User prompts sanitized

    Best Practices¶

    Template Design¶

    1. Use consistent frontmatter structure
    2. Include all required DRUIDS fields
    3. Provide helpful prompts
    4. Set cursor positions
    5. Include default values

    System Commands¶

    1. Validate all inputs
    2. Handle errors gracefully
    3. Use timeouts appropriately
    4. Log command execution
    5. Avoid side effects

    Performance¶

    1. Minimize system calls
    2. Cache repeated values
    3. Use async operations
    4. Limit template complexity
    5. Test on mobile devices

    Troubleshooting¶

    Template Not Triggering¶

    1. Check folder mapping
    2. Verify file creation location
    3. Ensure template exists
    4. Check console for errors

    System Command Failing¶

    1. Test command in terminal
    2. Check path variables
    3. Verify permissions
    4. Review timeout settings

    Cursor Position Issues¶

    1. Use tp.file.cursor()
    2. Number cursors sequentially
    3. Test cursor jumping
    4. Check syntax highlighting
