    Obsidian Tasks Plugin Configuration¶

    Overview¶

    The Tasks plugin has been configured to support the DRUIDS democratic centralist workflow with custom statuses that map to organizational processes.

                ↓
            ⏸️ Blocked

    Task Format Examples¶

    Basic Task¶

    - [ ] Implement security audit script #task 📅 2025-07-15 🏷️ security

    Democratic Workflow Task¶

    - [🗳️] Vote on new member application #vote 📅 2025-07-01 👥 @membership-committee

    Blocked Task¶

    - [⏸️] Deploy new infrastructure #blocked ⛔ awaiting-security-review

    Multi-stage Task¶

    - [🤝] Implement proposal PROP-2025-001 #implementation
  - [ ] Create implementation plan
  - [ ] Review with committee
  - [🎯] Validate approach
  - [ ] Execute implementation
  - [📝] Document outcomes

    Query Examples¶

    Active Proposals in Voting¶

    TASK
WHERE status = "🗳️"
GROUP BY file.folder

    Blocked Tasks Report¶

    TASK
WHERE status = "⏸️"
SORT priority DESC

    Consensus Achieved¶

    TASK
WHERE status = "🤝"
AND file.folder = "proposals"

    Global Filter Settings¶

    {
  "globalFilter": "#task",
  "removeGlobalFilter": true,
  "taskFormat": "tasksPluginEmoji"
}

    Purpose: Only tracks items explicitly marked with #task to avoid cluttering with regular checkboxes.

    Date Handling¶

    Configured Date Triggers¶

    • 📅 - Due date
    • ⏳ - Scheduled date
    • ✅ - Done date
    • ➕ - Created date
    • 🗳️ - Vote date (custom)

    Date Format¶

    "dueDateFormat": "YYYY-MM-DD"

    Recurrence Patterns¶

    Weekly Meeting Tasks¶

    - [ ] Prepare weekly meeting agenda 🔁 every week on Monday 📅 2025-07-01

    Monthly Reviews¶

    - [ ] Security audit review 🔁 every month on the 1st 📅 2025-07-01

    Integration with Other Plugins¶

    Kanban Integration¶

    Tasks automatically appear in Kanban boards based on status:

    • 📋 → Backlog column
    • 🚧 → In Progress column
    • 🗳️ → Voting column
    • ✅ → Done column

    Dataview Queries¶

    // All voting tasks
dv.taskList(dv.pages().file.tasks
  .where(t => t.symbol === "🗳️"))

// Blocked tasks by assignee
dv.taskList(dv.pages().file.tasks
  .where(t => t.symbol === "⏸️")
  .sort(t => t.text))

    Status Groups¶

    • Active: Todo, In Progress, Voting
    • Completed: Consensus, Validated, Documented, Done
    • Blocked: Blocked only

    Best Practices¶

    Task Creation¶

    1. Always include #task tag
    2. Add due dates for time-sensitive items
    3. Use status symbols consistently
    4. Link to parent proposals/meetings
    5. Include assignee for accountability

    Task Management¶

    1. Update status promptly
    2. Document blocking reasons
    3. Link related tasks
    4. Use recurrence for regular tasks
    5. Archive completed tasks monthly

    Query Optimization¶

    1. Use folder restrictions in queries
    2. Limit date ranges for performance
    3. Index frequently queried fields
    4. Cache complex query results

    Troubleshooting¶

    Tasks Not Appearing¶

    • Verify #task tag is present
    • Check global filter settings
    • Ensure proper task format

    Status Not Updating¶

    • Use command palette for status changes
    • Verify custom status symbols
    • Check for conflicting plugins

    Performance Issues¶

    • Reduce number of open task queries
    • Archive old completed tasks
    • Limit recursive folder searches
