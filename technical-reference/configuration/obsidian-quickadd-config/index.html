    Obsidian QuickAdd Plugin Configuration¶

    Overview¶

    QuickAdd provides rapid workflow automation through macros, templates, and captures. Configured for DRUIDS democratic centralist workflows and security operations.

    ⚠️ Security Alert¶

    Critical: API keys must NEVER be stored in QuickAdd configuration. The apiKey field should always be empty:

    "apiKey": ""

    Configured Macros¶

    1. Security Classification Review¶

    Purpose: Validates document security classification matches vault location

    Configuration:

    {
  "name": "Security Classification Review",
  "commands": [
    {
      "type": "Capture",
      "captureToActiveFile": true,
      "format": "Security Review: {{date:YYYY-MM-DD HH:mm}}\n- Vault: {{vault}}\n- Classification: {{value:security_level}}\n- Status: {{value:validation_status}}"
    }
  ]
}

    Usage:

    1. Run from any document
    2. Confirms security level matches vault
    3. Logs review with timestamp

    2. Vote Tracker¶

    Purpose: Records voting outcomes for democratic decisions

    Configuration:

    {
  "name": "Vote Tracker",
  "commands": [
    {
      "type": "Template",
      "templatePath": "templates/vote-record.md",
      "fileNameFormat": "votes/{{date:YYYY-MM-DD}}-{{value:proposal_id}}"
    }
  ]
}

    Fields Captured:

    • Proposal ID
    • Vote date/time
    • Participants
    • Results (for/against/abstain)
    • Decision outcome
    • Next steps

    3. Action Item¶

    Purpose: Creates trackable tasks from meetings or decisions

    Configuration:

    {
  "name": "Action Item",
  "commands": [
    {
      "type": "Multi",
      "commands": [
        {
          "type": "Capture",
          "captureToActiveFile": false,
          "createFileIfNotExists": true,
          "format": "- [ ] {{value:action}} #task\n  Assigned: {{value:assignee}}\n  Due: {{value:due_date}}\n  From: [{{activeFile}}]({{activeFile}}.md)",
          "captureTo": "tasks/{{date:YYYY-MM}}-tasks.md"
        },
        {
          "type": "Capture",
          "captureToActiveFile": true,
          "format": "\n- [ ] {{value:action}} → [Task List](../tasks/{{date:YYYY-MM}}-tasks.md)"
        }
      ]
    }
  ]
}

    Workflow:

    1. Creates task in central task list
    2. Adds reference in current document
    3. Maintains bidirectional links

    Template Configurations¶

    Meeting Template¶

    {
  "name": "New Meeting",
  "type": "Template",
  "templatePath": "templates/meeting-template.md",
  "fileNameFormat": "meetings/{{date:YYYY-MM-DD}}-{{value:meeting_type}}",
  "createInFolder": "meetings"
}

    Proposal Template¶

    {
  "name": "New Proposal",
  "type": "Template", 
  "templatePath": "templates/proposal-template.md",
  "fileNameFormat": "proposals/{{value:proposal_type}}-{{date:YYYY-MM-DD}}",
  "openFile": true
}

    Capture Configurations¶

    Quick Note¶

    {
  "name": "Quick Note",
  "type": "Capture",
  "format": "## {{date:HH:mm}} - {{value:title}}\n{{value:content}}\n#quicknote",
  "captureTo": "daily/{{date:YYYY-MM-DD}}.md",
  "appendLink": true
}

    CSC Entry¶

    {
  "name": "CSC Entry",
  "type": "Capture",
  "format": "### Criticism/Self-Criticism - {{date:YYYY-MM-DD}}\n**Type**: {{value:csc_type}}\n**Topic**: {{value:topic}}\n**Reflection**: {{value:reflection}}\n**Action Items**: {{value:actions}}",
  "captureTo": "csc/{{date:YYYY-MM}}-csc-log.md"
}

    Multi-Command Workflows¶

    Complete Proposal Workflow¶

    {
  "name": "Complete Proposal Workflow",
  "type": "Multi",
  "commands": [
    {
      "type": "Template",
      "templatePath": "templates/proposal-template.md"
    },
    {
      "type": "Capture",
      "captureTo": "proposals/index.md",
      "format": "- [{{fileName}}]({{fileName}}.md) - {{value:title}} ({{date:YYYY-MM-DD}})"
    },
    {
      "type": "Notification",
      "message": "Proposal created and indexed"
    }
  ]
}

    Hotkey Assignments¶

    Hotkey Macro Purpose
    Ctrl+Alt+S Security Review Quick security check
    Ctrl+Alt+V Vote Tracker Record vote outcome
    Ctrl+Alt+A Action Item Create task from meeting
    Ctrl+Alt+N Quick Note Rapid capture
    Ctrl+Alt+P New Proposal Start proposal workflow

    Integration with Other Plugins¶

    With Templater¶

    • QuickAdd can trigger Templater commands
    • Share template folder structure
    • Use Templater syntax in captures

    With Tasks¶

    • Creates properly formatted tasks
    • Uses consistent status symbols
    • Maintains task metadata

    With Dataview¶

    • Captures include Dataview fields
    • Structured for easy querying
    • Consistent metadata format

    Performance Optimization¶

    Settings¶

    {
  "enableSystemCommands": false,
  "suggestionsLimit": 5,
  "formatSyntax": "markdown"
}

    Rationale:

    • System commands disabled for security
    • Limited suggestions for speed
    • Markdown format for compatibility

    Security Best Practices¶

    DO NOT¶

    • Store API keys or credentials
    • Enable system commands without review
    • Create macros that access external services
    • Use user input in file paths without validation

    DO¶

    • Validate all user inputs
    • Use template sandboxing
    • Limit file system access
    • Audit macro configurations regularly

    Troubleshooting¶

    Macro Not Working¶

    1. Check command sequence
    2. Verify template paths
    3. Test each command individually
    4. Check console for errors

    Template Not Found¶

    1. Verify path is relative to vault root
    2. Check template file exists
    3. Ensure proper file extension
    4. Test with absolute path

    Capture Failing¶

    1. Verify destination folder exists
    2. Check file name format validity
    3. Ensure write permissions
    4. Test with simple format first
