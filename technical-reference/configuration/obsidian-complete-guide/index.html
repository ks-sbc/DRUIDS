    Complete Obsidian Configuration Guide¶

    Overview¶

    This comprehensive guide consolidates all Obsidian configuration information for DRUIDS. It covers core settings, essential plugins, security configurations, and workflow automations in one authoritative reference.

    Quick Start Configuration¶

    1. Essential Settings¶

    # Settings → Options → Editor
Vim key bindings: false  # Unless you're comfortable with Vim
Spell check: true
Default view mode: Source mode

# Settings → Options → Files & Links  
Default location for new notes: In the folder specified below
New note folder: ./inbox
Link format: Relative path
Use [[Wikilinks]]: true

# Settings → Options → Appearance
Theme: Obsidian default (or DRUIDS custom)
CSS snippets folder: .obsidian/snippets

    2. Core Plugin Installation Order¶

    Install plugins in this sequence to avoid conflicts:

    1. Obsidian Git - Version control (install first)
    2. Templater - Advanced templates
    3. Dataview - Dynamic queries
    4. Tasks - Task management
    5. Style Settings - Visual customization
    6. Fort Knox - Security enforcement (custom plugin)

    Branch-Specific Settings¶

    # Main branch (L0 - Public)
Auto-sync: Enabled
Auto-push: Enabled
Commit frequency: Every 15 minutes

# L1-internal branch (Member)
Auto-sync: Enabled
Auto-push: Consider manual
Commit frequency: Every 30 minutes

# L2-secure branch (Cadre)
Auto-sync: Disabled
Auto-push: NEVER
Commit: Manual only

# Common queries for DRUIDS

    // Active tasks by security level
table security, status, due
from #task and -"_templates"
where !completed
sort security desc, due asc

// Recent meetings
table date, attendees, decisions
from #meeting
sort date desc
limit 10

// Pending proposals
table status, created, vote-date
from #proposal
where status != "implemented"
sort created desc

    Automation Workflows¶

    QuickAdd Macros¶

    // Create Meeting macro
module.exports = async (params) => {
  const {quickAddApi: {inputPrompt, suggester}} = params;

  const meetingType = await suggester(
    ["Weekly Team", "Planning", "Emergency", "CSC"],
    ["weekly", "planning", "emergency", "csc"]
  );

  const date = window.moment().format("YYYY-MM-DD");
  const time = await inputPrompt("Meeting time (HH:MM)");

  const fileName = `meetings/${date}-${meetingType}-meeting`;
  const template = `_templates/meeting-${meetingType}.md`;

  await params.quickAddApi.createFileFromTemplate(
    template, 
    fileName,
    true
  );
};

    Performance Optimization¶

    Plugin Load Order¶

    Optimize startup by controlling plugin load order:

    1. Core functionality (Git, Templater)
    2. Data processing (Dataview, Tasks)
    3. UI enhancements (Style Settings, Kanban)
    4. Optional features (Advanced Tables, Slides)

    Disabled by Default¶

    These plugins should be enabled only when needed:

    • Graph View - Resource intensive
    • Backlinks - Can slow large vaults
    • Page Preview - Memory usage
    • Audio Recorder - Rarely used

    Troubleshooting¶

    Common Issues¶

    Git Authentication Failed¶

    # Check SSH key
ssh -T git@github.com

# Update Git credentials
git config credential.helper manager

    Plugin Conflicts¶

    1. Disable all plugins
    2. Enable core plugins one by one
    3. Test after each activation
    4. Document conflicts

    Performance Issues¶

    # Check vault size
du -sh .

# Find large files
find . -type f -size +10M

# Clean workspace
rm .obsidian/workspace.json

    Security Audit Commands¶

    # Check for missing security classifications
rg -l "^---" . | xargs rg -L "security:"

# Find cross-tier links
rg "\[\[.*L2.*\]\]" vault/public/

# Verify document IDs
node scripts/validate-document-ids.js

    Migration Checklist¶

    When setting up a new vault:

    • Install Obsidian
    • Clone appropriate repository
    • Install plugins in order
    • Copy security CSS snippets
    • Configure Git plugin
    • Set up templates folder
    • Configure Templater
    • Enable security snippets
    • Test Git sync
    • Configure hotkeys
    • Run security audit
    • Document customizations

    Best Practices¶

    Daily Workflow¶

    1. Pull changes on startup
    2. Use templates for new documents
    3. Commit with meaningful messages
    4. Review security classifications
    5. Push changes before closing

    Weekly Maintenance¶

    1. Review and clean tags
    2. Update task statuses
    3. Archive completed items
    4. Check for plugin updates
    5. Run security audit

    Monthly Reviews¶

    1. Analyze vault statistics
    2. Optimize performance
    3. Update documentation
    4. Review automation rules
    5. Backup configurations
