    Obsidian Table Plugins Configuration Guide¶

    Overview¶

    This guide covers the table management plugins (Table Editor and Notion-like Tables) configured for DRUIDS data organization and workflow tracking.

    Table Editor Plugin¶

    Purpose¶

    Provides enhanced markdown table editing with templates, formatting, and keyboard shortcuts optimized for meeting minutes and organizational data.

    Pre-Configured Table Templates¶

    1. Voting Record¶

    | Member | Vote | Comments | Time |
|--------|:----:|----------|:----:|
| Alice  | For  | Supports proposal fully | 14:32 |
| Bob    | Against | Concerns about resources | 14:35 |
| Charlie | Abstain | Need more information | 14:37 |

    Column Settings:

    • Member: 150px, left-aligned
    • Vote: 80px, center-aligned
    • Comments: 200px, left-aligned
    • Time: 100px, center-aligned

    2. Action Items¶

    | Task | Assignee | Due Date | Status | Priority |
|------|----------|:--------:|:------:|:--------:|
| Draft proposal | Alice | 2025-07-15 | 🚧 In Progress | P1 |
| Review security | Bob | 2025-07-10 | ⏸️ Blocked | P0 |
| Update docs | Charlie | 2025-07-20 | 📋 Todo | P2 |

    Usage: Track tasks from meetings with clear ownership and deadlines.

    3. Security Matrix¶

    | Document | Security Level | Classification | Last Review | Reviewer |
|----------|:--------------:|:--------------:|:-----------:|----------|
| Meeting-001 | L1 | Correct | 2025-06-30 | Security |
| Proposal-042 | L0 | Needs Review | 2025-06-15 | - |
| Report-007 | L2 | Encrypted | 2025-06-28 | Cadre |

    Purpose: Audit document security classifications.

    4. Meeting Attendance¶

    | Name | Role | Present | Voting Rights | Notes |
|------|------|:-------:|:-------------:|-------|
| Alice | Chair | ✓ | Yes | |
| Bob | Secretary | ✓ | Yes | Taking minutes |
| Charlie | Member | ✗ | No | Excused absence |

    5. Risk Assessment¶

    | Risk | Impact | Probability | Mitigation | Owner |
|------|:------:|:-----------:|------------|-------|
| Data breach | High | Low | Encryption, access control | Security |
| Member burnout | Medium | Medium | Task distribution | Organizer |

    Notion-like Tables Plugin¶

    Purpose¶

    Provides database-like functionality with advanced features like filtering, sorting, and multiple views for complex data management.

    Pre-Configured Database Templates¶

    1. Task Tracker¶

    Columns:

    • Task (text)
    • Status (select): Todo, In Progress, Done, Blocked
    • Priority (select): P0-P5
    • Assignee (person)
    • Due Date (date)
    • Complete (checkbox)
    • Notes (text)

    Use Case: Comprehensive task management with filtering and views.

    2. Meeting Minutes¶

    Columns:

    • Agenda Item (text)
    • Discussion (text)
    • Decision (text)
    • Action Items (text)
    • Owner (person)
    • Due Date (date)

    Features: Track meeting outcomes and follow-up actions.

    3. Proposal Tracker¶

    Columns:

    • Proposal ID (text)
    • Title (text)
    • Proposer (person)
    • Status (select): Draft, Discussion, Voting, Approved, Rejected, Implemented
    • Committee (select): General, Membership, Education, Security, Finance
    • Vote Date (date)
    • For/Against/Abstain (numbers)
    • Links (file)

    Purpose: Complete proposal lifecycle management.

    4. Security Audit¶

    Columns:

    • Document (text)
    • Security Level (select): L0, L1, L2
    • Classification Valid (checkbox)
    • Location (text)
    • Last Audit (date)
    • Issues (multi-select): Misclassified, Missing Metadata, Wrong Vault, Needs Encryption
    • Auditor (person)

    Usage: Systematic security compliance tracking.

    5. Member Directory¶

    Columns:

    • Name (text)
    • Role (select): Member, Organizer, Committee Chair, Secretary, Treasurer
    • Committee (multi-select): All committees
    • Join Date (date)
    • Status (select): Active, Inactive, Leave
    • Contact (text)
    • Notes (text)

    Filter Presets¶

    High Priority Tasks¶

    Filters:
- Priority is P0 or P1
- Status is not Done

    My Tasks¶

    Filters:
- Assignee is @me
- Status is not Done

    Blocked Items¶

    Filters:
- Status is Blocked

    Active Proposals¶

    Filters:
- Status is Discussion or Voting

    View Presets¶

    1. Kanban by Status¶

    • Type: Board view
    • Group by: Status
    • Shows: Task cards in columns

    2. Calendar by Due Date¶

    • Type: Calendar view
    • Date field: Due Date
    • Shows: Tasks on calendar

    3. Gallery View¶

    • Type: Gallery
    • Properties: Title, Status, Priority
    • Shows: Visual cards

    Formula Examples¶

    Days Until Due¶

    dateBetween(prop("Due Date"), now(), "days")

    Vote Result¶

    if(prop("For") > prop("Against"), "Passed", "Failed")

    Task Age¶

    dateBetween(now(), prop("Created"), "days") + " days"

    Integration Patterns¶

    Table Editor + Markdown¶

    • Quick table creation
    • In-line editing
    • Preserve formatting
    • Export to plain text

    Notion-like Tables + Dataview¶

    • Query table data
    • Create dynamic views
    • Aggregate calculations
    • Cross-reference data

    Tables + Tasks Plugin¶

    • Task tables with status
    • Due date tracking
    • Assignment management
    • Progress visualization

    Best Practices¶

    Table Selection Guide¶

    Use Table Editor when:

    • Creating simple markdown tables
    • Need quick formatting
    • Working with meeting minutes
    • Want keyboard shortcuts
    • Prefer plain text storage

    Use Notion-like Tables when:

    • Need database features
    • Want multiple views
    • Require filtering/sorting
    • Managing complex data
    • Need formulas

    Data Organization¶

    1. Consistent Headers: Use same names across tables
    2. Data Types: Choose appropriate column types
    3. Naming Convention: Clear, descriptive names
    4. Relationships: Link related tables
    5. Regular Cleanup: Archive old data

    Performance Tips¶

    1. Limit Rows: Keep under 500 per table
    2. Index Key Columns: Use select/date for filtering
    3. Archive Old Data: Move completed items
    4. Optimize Formulas: Avoid complex calculations
    5. Use Views: Instead of duplicate tables

    Common Workflows¶

    Meeting Minutes Workflow¶

    1. Create table with Table Editor template
    2. Fill attendance during meeting
    3. Add action items in real-time
    4. Export to Notion-like table for tracking
    5. Follow up on assignments

    Project Task Management¶

    1. Create Notion-like task database
    2. Add all project tasks
    3. Assign and prioritize
    4. Use Kanban view for status
    5. Filter for weekly reviews

    Security Audit Process¶

    1. Generate audit table monthly
    2. List all documents
    3. Check classifications
    4. Mark issues found
    5. Export report

    Troubleshooting¶

    Table Editor Issues¶

    Formatting Lost

    • Check auto-format settings
    • Verify markdown mode
    • Use monospace font

    Shortcuts Not Working

    • Check keybinding conflicts
    • Verify plugin enabled
    • Restart Obsidian

    Notion-like Tables Issues¶

    Performance Slow

    • Reduce table size
    • Disable unused columns
    • Clear filter cache
    • Use pagination

    Views Not Updating

    • Refresh database
    • Check filter syntax
    • Verify column names
    • Clear cache

    Data Loss Prevention

    • Regular exports
    • Version control
    • Backup databases
    • Test before bulk edits

    Advanced Features¶

    CSV Import/Export¶

    Import Settings:

    {
  "delimiter": ",",
  "treatFirstRowAsHeader": true,
  "trimWhitespace": true
}

    Export Settings:

    {
  "includeHeaders": true,
  "dateFormat": "YYYY-MM-DD"
}

    Custom Formulas¶

    // Meeting attendance rate
prop("Present") / prop("Total") * 100 + "%"

// Task completion status  
if(prop("Complete"), "✅", 
  if(prop("Due Date") < now(), "⚠️ Overdue", "🔄 Pending"))

// Priority scoring
switch(prop("Priority"),
  "P0", 100,
  "P1", 80,
  "P2", 60,
  "P3", 40,
  "P4", 20,
  "P5", 10)
