    Essential Obsidian Settings¶

    Overview¶

    This guide documents the core Obsidian settings required for DRUIDS vault configuration. These settings ensure consistency across all vaults and support democratic centralist workflows.

    Editor Settings¶

    Basic Configuration¶

    Settings → Editor:
  Readable line length: ON
  Strict line breaks: OFF
  Show frontmatter: ON
  Show line numbers: OFF
  Use tabs: OFF
  Tab size: 2
  Vim key bindings: OFF (unless experienced)
  Spell check: ON

    Advanced Configuration¶

    Settings → Editor (Advanced):
  Auto pair brackets: ON
  Auto pair Markdown syntax: ON
  Smart indent lists: ON
  Fold heading: ON
  Fold indent: ON
  Default view mode for new tabs: Source mode
  Default editing mode: Live Preview

    Files & Links¶

    Core Settings¶

    Settings → Files & Links:
  Confirm file deletion: ON
  Deleted files: Move to Obsidian trash
  Always update internal links: ON
  New link format: Shortest path when possible
  Use [[Wikilinks]]: ON
  Default location for new notes: In the folder specified below
  New note folder: ./inbox
  Default location for new attachments: In subfolder under current folder
  Attachment folder path: ./attachments

    Link Settings¶

    Settings → Files & Links (continued):
  Automatically update internal links: ON
  Detect all file extensions: OFF
  Excluded files: .git, .obsidian, node_modules

    Appearance¶

    Theme Settings¶

    Settings → Appearance:
  Base theme: Dark (recommended for security indicators)
  Theme: Default or DRUIDS custom
  Translucency: OFF (security consideration)
  Show inline title: ON
  Show tab title bar: ON
  Font size: 16
  Quick font size adjustment: ON

    CSS Snippets¶

    Settings → Appearance → CSS snippets:
  Enable snippets folder: .obsidian/snippets

Required snippets:
  - security-indicators.css: ON
  - druids-theme.css: ON (optional)
  - accessibility-patterns.css: ON
  - print-security.css: ON

    Hotkeys¶

    Essential Shortcuts¶

    Settings → Hotkeys:
  Quick switcher: Ctrl/Cmd+O
  Command palette: Ctrl/Cmd+P
  Search in all files: Ctrl/Cmd+Shift+F
  Open settings: Ctrl/Cmd+,

Security-specific:
  Insert template: Alt+T
  Open daily note: Ctrl/Cmd+D
  Toggle edit/preview: Ctrl/Cmd+E

    Core Plugins¶

    Required Core Plugins¶

    Settings → Core plugins:
  File explorer: ON
  Search: ON
  Quick switcher: ON
  Graph view: ON
  Backlinks: ON
  Outgoing links: ON
  Tags: ON
  Page preview: ON
  Templates: ON
  Command palette: ON
  Markdown importer: ON
  Word count: ON
  File recovery: ON

    Optional Core Plugins¶

    Settings → Core plugins (optional):
  Audio recorder: User preference
  Canvas: OFF (not used in DRUIDS)
  Daily notes: User preference
  Random note: OFF
  Slash commands: User preference
  Bookmarks: User preference

    Security Settings¶

    Vault Security¶

    Settings → About → Advanced:
  Safe mode: OFF (required for community plugins)

Settings → Community plugins:
  Restricted mode: OFF
  Browse community plugins: Allowed

Security considerations:
  - Only install verified plugins
  - Review plugin permissions
  - Regular security audits

    File Recovery¶

    Settings → File recovery:
  Snapshot interval: 5 minutes
  Keep snapshots for: 7 days

Note: Not a replacement for Git backups

    Sync Settings¶

    Git Integration¶

    For Obsidian Git plugin:
  Disable built-in sync
  Use Git for version control
  See Git plugin configuration

    Mobile Settings¶

    Settings → Mobile:
  Configure toolbar: Customize for workflow
  Pull down to sync: OFF (use Git)
  Always show keyboard: User preference

    Performance Settings¶

    Optimization¶

    Settings → Editor:
  Auto-complete: User preference
  Context menu: ON

Settings → Files & Links:
  Always update internal links: ON (may impact large vaults)

Graph view settings:
  Reduce nodes for performance
  Limit depth of connections

    Language & Region¶

    Settings → About:
  Language: English (or organizational preference)
  Interface language: Match system

Settings → Editor:
  Spell check language: English (US/UK per preference)

    Vault-Specific Settings¶

    Public Vault (L0)¶

    Specific configurations:
  - New note folder: ./inbox/public
  - Attachment folder: ./attachments/public
  - Default template: public-note-template

    Member Vault (L1)¶

    Specific configurations:
  - New note folder: ./inbox/member
  - Attachment folder: ./attachments/member
  - Default template: member-note-template
  - Additional security CSS enabled

    Cadre Vault (L2)¶

    ⚠️ FORBIDDEN - No AI/LLM access to L2 configurations
Manual configuration only by authorized users

    Settings Export/Import¶

    Backup Settings¶

    # Export settings
cp -r .obsidian/config .obsidian/config.backup

# Key files to backup:
.obsidian/config
.obsidian/appearance.json
.obsidian/core-plugins.json
.obsidian/community-plugins.json
.obsidian/hotkeys.json

    Share Settings¶

    To share settings across vaults:
1. Use symlinks from shared/obsidian
2. Version control .obsidian folder
3. Document any vault-specific changes

    Troubleshooting¶

    Common Issues¶

    1. Settings not saving
    2. Check file permissions
    3. Verify .obsidian folder exists

    4. Restart Obsidian

    5. Plugins not loading

    6. Verify safe mode is OFF
    7. Check plugin compatibility

    8. Review console for errors

    9. Performance issues

    10. Reduce file recovery frequency
    11. Disable unused core plugins
    12. Optimize graph view settings

    Best Practices¶

    1. Regular Backups: Export settings before major changes
    2. Documentation: Note any custom configurations
    3. Consistency: Keep settings aligned across vaults
    4. Security: Regular review of plugin permissions
    5. Performance: Monitor vault size and adjust settings
