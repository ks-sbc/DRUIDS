    CSS Snippets Configuration¶

    Overview¶

    CSS snippets provide custom styling for DRUIDS vaults, primarily implementing the visual security indicator system. The "custom-css" entry in community plugins refers to Obsidian's built-in CSS snippet functionality.

    Snippet Location¶

    All CSS snippets are stored in: 

    .obsidian/snippets/

    Snippets must have .css extension to be recognized by Obsidian.

    Core DRUIDS Snippets¶

    1. Security Indicators (security-indicators.css)¶

    Implements the three-tier visual security system:

    /* L0 - Public (Forest Green #228B22) */
.L0, 
[data-security="L0"],
.nav-file[data-path*="/public/"],
.tree-item[data-path*="/public/"] {
  border-left: 4px solid #228B22 !important;
  background: linear-gradient(
    45deg, 
    transparent 25%, 
    rgba(34, 139, 34, 0.05) 25%, 
    rgba(34, 139, 34, 0.05) 50%, 
    transparent 50%
  );
  background-size: 20px 20px;
}

/* L1 - Member (Saddle Brown #8B4513) */
.L1,
[data-security="L1"], 
.nav-file[data-path*="/member/"],
.tree-item[data-path*="/member/"] {

    -45deg,

    2. DRUIDS Theme (druids-theme.css)¶

    Revolutionary aesthetic theme:

    /* Color Variables */
body.theme-dark {
  --druids-primary: #228B22;        /* Forest green */
  --druids-secondary: #8B4513;      /* Saddle brown */
  --druids-background: #1B3B1B;    /* Dark forest */
  --druids-text: #E6E6FA;          /* Lavender */
  --druids-accent: #FFD700;        /* Gold */
}

/* Industrial Typography */
body {
  --font-text: "JetBrains Mono", "Fira Code", monospace;
  --font-ui: "Inter", "Roboto", sans-serif;
  --font-monospace: "JetBrains Mono", "Consolas", monospace;
}

/* Revolutionary Headers */
h1, h2, h3, h4, h5, h6 {
  font-family: var(--font-ui);
  font-weight: 900;
  text-transform: uppercase;
  letter-spacing: 0.05em;
}

h1 { color: var(--druids-primary); }
h2 { color: var(--druids-secondary); }
h3 { color: var(--druids-accent); }

    High-contrast patterns for colorblind users:

.nav-file[data-path*="/public/"]::after {
  content: "";
  position: absolute;
  right: 8px;
  color: #228B22;
  font-weight: bold;
}

/* Diagonal lines for L1 */
.nav-file[data-path*="/member/"]::after {
  content: "///";
  position: absolute;
  right: 8px;
  color: #8B4513;
  font-weight: bold;

/* Cross pattern for L2 */
.nav-file[data-path*="/cadre/"]::after {
  content: "XXX";
  position: absolute;
  right: 8px;
  color: #A0522D;
  font-weight: bold;
}

    4. Print Styles (print-security.css)¶

    Security indicators for printed documents:

    @media print {
  /* Add security classification to printed pages */
  .markdown-preview-view::before {
    content: attr(data-security-level) " - DRUIDS";
    position: fixed;
    top: 10mm;
    right: 10mm;
    font-size: 10pt;
    font-weight: bold;
    text-transform: uppercase;
  }

  /* L0 Print Header */
  [data-security="L0"] .markdown-preview-view::before {
    color: #228B22;
    content: "PUBLIC - L0 - DRUIDS";
  }

  /* L1 Print Header */
  [data-security="L1"] .markdown-preview-view::before {
    color: #8B4513;
    content: "MEMBER ONLY - L1 - DRUIDS";
  }

  /* L2 Print Header */
  [data-security="L2"] .markdown-preview-view::before {
    color: #A0522D;
    content: "CADRE ONLY - L2 - DRUIDS";
  }
}

    Enabling Snippets¶

    1. Place .css files in .obsidian/snippets/
    2. Settings Appearance CSS snippets
    3. Toggle on desired snippets
    4. Use "Reload" button to refresh

    Custom Snippet Template¶

    /* Snippet: snippet-name.css
 * Purpose: Brief description
 * Author: Your name
 * Date: YYYY-MM-DD
 * Security: L0/L1/L2
 */

/* Variables */
:root {
  --custom-var: value;
}

/* Dark theme overrides */
body.theme-dark {
  --custom-var: dark-value;
}

/* Light theme overrides */
body.theme-light {
  --custom-var: light-value;
}

/* Main styles */
.custom-class {
  property: value;
}

    Best Practices¶

    Performance¶

    1. Minimize use of !important
    2. Avoid complex selectors
    3. Use CSS variables for consistency
    4. Group related styles

    Security¶

    1. Don't hide security indicators
    2. Ensure contrast meets WCAG standards
    3. Test with colorblind simulators
    4. Maintain visual hierarchy

    Organization¶

    1. One snippet per feature
    2. Clear naming convention
    3. Document complex selectors
    4. Version control snippets

    Common Customizations¶

    Hide UI Elements¶

    /* Hide vault name (OpSec) */
.nav-folder-title[data-path="/"] .nav-folder-title-content::before {
  content: "Vault";
}

/* Hide status bar items */
.status-bar-item.plugin-word-count {
  display: none;
}

    Debugging Snippets¶

    Developer Tools¶

    1. Ctrl/Cmd + Shift + I to open DevTools
    2. Inspect element to find selectors
    3. Test changes in Elements panel
    4. Copy working CSS to snippet file

    Common Issues¶

    • Not loading: Check file extension is .css
    • Not applying: Verify snippet is enabled
    • Conflicts: Check specificity and order
    • Performance: Reduce complex selectors
