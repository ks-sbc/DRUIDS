    Security Configuration Guide¶

    Overview¶

    This guide documents the security configuration for DRUIDS Obsidian vaults, implementing the three-tier security model with visual indicators, access controls, and audit capabilities.

    Three-Tier Security Model¶

    Security Levels¶

    Level Classification Color Pattern Access
    L0 Public Forest Green (#228B22) Dots Everyone
    L1 Member Saddle Brown (#8B4513) Diagonal lines Members only
    L2 Cadre Sienna (#A0522D) Cross-hatch Leadership only

    Visual Implementation¶

    /* Applied via security-indicators.css */
L0: Green border + dot pattern background
L1: Brown border + diagonal line background  
L2: Sienna border + cross-hatch background

    Vault Structure¶

    Directory Organization¶

    /vault/
 public/          # L0 - Public content
    guides/
    resources/
    statements/
 member/          # L1 - Member content
    meetings/
    proposals/
    projects/
 cadre/           # L2 - FORBIDDEN to AI/LLMs
     [REDACTED]

    Security Enforcement¶

    1. Git Repositories: Separate repos per security level
    2. Visual Indicators: CSS-based security badges
    3. Plugin Validation: Fort Knox plugin for compliance
    4. Audit Trail: Git history as security log

    Plugin Security Settings¶

    GPGCrypt Configuration¶

    {
  "keyId": "",
  "method": "gpg",
  "binary": "/usr/bin/gpg",
  "armor": true,
  "alwaysEncrypt": false,
  "alwaysDecrypt": true,
  "trustModel": "always",
  "defaultRecipient": "",
  "encryptionAlgorithm": "AES256",
  "smartcardSupport": true,
  "alwaysSign": true,
  "requireStrongKey": true,
  "minKeyLength": 4096
}

    Linter Security Rules¶

    {
  "customRules": [
    {
      "name": "Security Level Required",
      "regex": "^---[\\s\\S]*?security:\\s*(L0|L1|L2)[\\s\\S]*?---",
      "error": "Missing security classification in frontmatter"
    },
    {
      "name": "Document ID Required",
      "regex": "^---[\\s\\S]*?document_id:\\s*[\\w-]+[\\s\\S]*?---",
      "error": "Missing document_id in frontmatter"
    },
    {
      "name": "Vault Path Match",
      "regex": "^---[\\s\\S]*?security:\\s*L0[\\s\\S]*?---",
      "flags": "m",
      "error": "L0 documents must be in /public/ folder"
    }
  ]
}

    Fort Knox Plugin Settings¶

    Custom security enforcement plugin:

    {
  "enforceSecurityClassification": true,
  "blockCrossTierLinks": true,
  "requireFrontmatter": true,
  "auditMode": false,
  "preventL2Access": true,  // Block all L2 operations
  "allowedFields": [
    "security",
    "document_id", 
    "type",
    "version"
  ],
  "validSecurityLevels": ["L0", "L1", "L2"],
  "pathValidation": {
    "L0": "/public/",
    "L1": "/member/",
    "L2": "/cadre/"
  }
}

    Access Control¶

    Git-Based Security¶

    Public Vault (L0):
  - Public repository
  - No authentication required
  - Read-only for anonymous users

Member Vault (L1):
  - Private repository
  - Authentication required
  - Member group access

Cadre Vault (L2):
  - Separate private repository
  - Multi-factor authentication
  - Leadership group only
  - NO AI/LLM ACCESS

    Link Security¶

    // Fort Knox plugin prevents cross-tier links
if (linkTarget.security > currentDoc.security) {
  blockLink("Cannot link to higher security tier");
}

    Audit Configuration¶

    Git Commit Hooks¶

    # .git/hooks/pre-commit
#!/bin/bash
# Validate security classifications
node /shared/scripts/security-validator.js

    Audit Queries¶

    // Find misclassified documents
TABLE security, file.folder
FROM ""
WHERE security = "L0" AND !contains(file.path, "/public/")
SORT file.mtime DESC

    Security Reports¶

    // Security distribution
TABLE 
  length(rows) as Count
FROM ""
GROUP BY security

    Mobile Security¶

    iOS/Android Settings¶

    Obsidian Mobile:
  - Biometric authentication: REQUIRED
  - Auto-lock: 5 minutes
  - Disable screenshots: L2 vaults
  - Local storage only: L2 vaults

    Sync Restrictions¶

    L0 Vaults:
  - Git sync allowed
  - Public cloud backup OK

L1 Vaults:  
  - Git sync with auth
  - Encrypted backups only

L2 Vaults:
  - NO cloud sync
  - Local Git only
  - Air-gapped preferred

    Security Workflows¶

    Document Creation¶

    1. Choose appropriate vault
    2. Use security-aware template
    3. Verify frontmatter security field
    4. Check visual indicators
    5. Commit with security context

    Security Review¶

    # Weekly security audit
./scripts/security-audit.sh

# Check for violations
rg -l "security: L[0-2]" | xargs -I {} sh -c 'echo "{}:$(grep -m1 "security:" {} )"'

    Incident Response¶

    1. Misclassification: Move file, update links, audit
    2. Unauthorized Access: Revoke keys, audit logs
    3. Cross-tier Leak: Immediate isolation, review

    Best Practices¶

    Do's¶

    • Always verify visual indicators
    • Use templates with pre-set security
    • Regular security audits
    • Separate vaults by security level
    • Train on security model

    Don'ts¶

    • Never ignore visual warnings
    • Don't copy between security tiers
    • No screenshots of L2 content
    • Avoid cross-tier links
    • Never share vault paths

    Troubleshooting¶

    Visual Indicators Not Showing¶

    1. Check CSS snippets enabled
    2. Verify snippet file names
    3. Reload Obsidian
    4. Check theme compatibility

    Security Validation Failures¶

    1. Verify frontmatter format
    2. Check file location
    3. Run manual validator
    4. Review Git hooks

    Access Issues¶

    1. Verify Git credentials
    2. Check group membership
    3. Confirm repository access
    4. Review authentication logs
