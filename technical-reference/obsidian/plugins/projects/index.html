    Projects Plugin Configuration¶

    Overview¶

    The Projects plugin provides project management capabilities with Dataview integration, custom views, and democratic centralist workflow support. It enables tracking of proposals, tasks, and security audits across DRUIDS infrastructure.

    Plugin Information¶

    • Plugin ID: projects
    • Version: 1.5.0
    • Purpose: Project tracking with dynamic views and Dataview queries
    • Security: Respects vault boundaries and security classifications

    Configuration¶

    Core Settings¶

    {
  "preferences": {
    "dateFormat": "YYYY-MM-DD",
    "showArchived": false,
    "projectSizeLimit": 2000,
    "communityProjectGallery": false,
    "defaultProjectPath": "Projects",
    "defaultTemplate": "templates/project-template.md",
    "showProjectPath": true,
    "projectCreationFolder": "Projects",
    "excludeFolders": [".trash", ".obsidian", "templates"],
    "includeSubfolders": true,
    "refreshInterval": 30
  }
}

    The plugin is configured with DRUIDS-specific fields:

    "defaultFields": [

    Pre-configured Projects¶

    1. DRUIDS Infrastructure Implementation¶

    ID: druids-implementation
    Path: Projects/DRUIDS
    Purpose: Track complete DRUIDS system implementation

    Views: - All Tasks: Table view of all DRUIDS-related tasks - Active Proposals: Democratic workflow proposals - Security Overview: Gallery grouped by security level - Progress Board: Kanban-style task board

    2. Democratic Centralist Workflow¶

    ID: democratic-workflow
    Path: Projects/Democratic-Workflow
    Purpose: Manage proposal, voting, and implementation cycles

    Views: - Current Proposals: Active proposals with voting status - Voting Queue: Calendar view of upcoming votes - Implementation Tracker: Post-approval task tracking

    3. Security Audit & Compliance¶

    ID: security-audit
    Path: Projects/Security
    Purpose: Monitor security classifications and compliance

    Views: - Document Security Matrix: All documents with security levels - Misclassified Documents: Documents in wrong directories - Audit Schedule: Calendar of security audits

    View Types¶

    Table View¶

    {
  "name": "View Name",
  "type": "table",
  "query": "dataview query here",
  "fields": ["field1", "field2", "field3"]
}

    Board View¶

    {
  "name": "Progress Board",
  "type": "board",
  "query": "task where project = \"ProjectName\"",
  "columns": ["Backlog", "In Progress", "Review", "Complete"]
}

    Calendar View¶

    {
  "name": "Schedule",
  "type": "calendar",
  "query": "from \"folder\" where date_field",
  "dateField": "date_field"
}

    Gallery View¶

    {
  "name": "Overview",
  "type": "gallery",
  "query": "from \"\" where grouping_field",
  "groupBy": "grouping_field"
}

    Dataview Integration¶

    The Projects plugin leverages Dataview for dynamic queries:

    Task Queries¶

    task where contains(project, "DRUIDS") and status != "complete"

    Document Queries¶

    from "proposals" 
where status = "active" and vote_date > date(today)
sort vote_date asc

    Security Audit Queries¶

    from "" 
where security_level and !contains(file.path, security_level)

    Best Practices¶

    Project Organization¶

    1. Use consistent project IDs following pattern: category-name-YYYY
    2. Create projects in designated folders by security level
    3. Link related proposals and tasks using [[]] syntax
    4. Regular status updates in project notes

    View Management¶

    1. Keep views focused on specific workflows
    2. Use meaningful view names
    3. Limit table fields to essential information
    4. Test Dataview queries before adding to views

    Security Considerations¶

    1. Ensure project paths respect vault boundaries
    2. Security level fields must match file location
    3. Exclude sensitive folders from project scanning
    4. Regular audits using security compliance views

    Common Issues¶

    Views Not Updating¶

    • Check Dataview plugin is enabled
    • Verify query syntax is correct
    • Ensure refresh interval is appropriate
    • Clear Dataview cache if needed

    Missing Projects¶

    • Verify project path is included in settings
    • Check folder isn't in exclude list
    • Ensure project has required frontmatter
    • Confirm file has .md extension

    Performance Issues¶

    • Reduce project size limit
    • Optimize Dataview queries
    • Increase refresh interval
    • Archive completed projects

    Integration with Other Plugins¶

    Tasks Plugin¶

    • Projects can display tasks using task queries
    • Task statuses sync with project boards
    • Due dates appear in calendar views

    Kanban Plugin¶

    • Board views complement Kanban boards
    • Different visualization for same data
    • Projects provide overview, Kanban for details

    Templater¶

    • Use Templater for dynamic project templates
    • Auto-generate project IDs
    • Insert current date and user
