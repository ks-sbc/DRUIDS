    Deploying DRUIDS Wiki to Cloudflare Pages¶

    This guide covers deploying the DRUIDS documentation to Cloudflare Pages with custom domain configuration.

    Prerequisites¶

    • GitHub repository with DRUIDS wiki code
    • Cloudflare account (free tier works)
    • Domain name (for custom subdomain)
    • Basic familiarity with Git and command line

    Step 1: Prepare Your Repository¶

    Required Files¶

    Ensure your repository has:

    1. mkdocs.yml - MkDocs configuration
    2. requirements.txt - Python dependencies
    3. docs/ directory - Your documentation content
    4. giscus.json - Comment security configuration

    Update Site URL¶

    Edit mkdocs.yml to set your production URL:

    site_url: https://druids.kssocialistbookclub.com

    Step 2: Create Cloudflare Pages Project¶

    Connect Repository¶

    1. Log into Cloudflare Dashboard
    2. Navigate to Workers & Pages
    3. Click Create application → Pages → Connect to Git
    4. Select your GitHub account and authorize access
    5. Choose the DRUIDS repository

    Configure Build Settings¶

    Set the following build configuration:

    Setting Value
    Project name druids-ksbc
    Production branch main
    Build command pip install -r requirements.txt && mkdocs build
    Build output directory site
    Root directory /

    Environment Variables¶

    Add these environment variables:

    PYTHON_VERSION = 3.11

    Click Save and Deploy to start the initial build.

    Step 3: Configure Custom Domain¶

    Add Domain in Cloudflare Pages¶

    1. Go to your Pages project → Settings → Custom domains
    2. Click Set up a custom domain
    3. Enter: druids.kssocialistbookclub.com
    4. Click Continue

    DNS Configuration¶

    Cloudflare will automatically create the CNAME record:

    Type: CNAME
Name: druids
Content: druids-ksbc.pages.dev
Proxy: ✓ Enabled

    SSL Certificate¶

    • SSL certificate is provisioned automatically
    • Usually takes < 5 minutes
    • Full end-to-end encryption enabled by default

    Step 4: GitHub Actions Workflow¶

    Create .github/workflows/cloudflare-deploy.yml:

    name: Deploy to Cloudflare Pages

on:
  push:
    branches: [main]
    paths:
      - 'docs/**'
      - 'mkdocs.yml'
      - 'requirements.txt'
      - 'overrides/**'
  pull_request:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  deployments: write
  pull-requests: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    name: Build and Deploy Documentation

    steps:
    - name: Checkout Repository
      uses: actions/checkout@v4
      with:
        fetch-depth: 0  # Full history for git plugins

    - name: Setup Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.11'
        cache: 'pip'
        cache-dependency-path: 'requirements.txt'

    - name: Cache MkDocs Files
      uses: actions/cache@v4
      with:
        path: .cache
        key: mkdocs-${{ hashFiles('requirements.txt') }}
        restore-keys: |
          mkdocs-

    - name: Install Dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Build Documentation
      run: mkdocs build --strict --verbose

    - name: Deploy to Cloudflare Pages
      id: cloudflare
      uses: cloudflare/wrangler-action@v3
      with:
        apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
        accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
        command: pages deploy site --project-name=druids-ksbc --branch=${{ github.head_ref || github.ref_name }}
        gitHubToken: ${{ secrets.GITHUB_TOKEN }}

    - name: Add Deploy Comment to PR
      if: github.event_name == 'pull_request'
      uses: actions/github-script@v7
      with:
        script: |
          const deploymentUrl = '${{ steps.cloudflare.outputs.deployment-url }}';
          github.rest.issues.createComment({
            issue_number: context.issue.number,
            owner: context.repo.owner,
            repo: context.repo.repo,
            body: `### 🚀 Documentation Preview\n\nYour changes have been deployed!\n\n**Preview URL:** ${deploymentUrl}\n\nThis preview will remain active until the PR is merged or closed.`
          });

    Step 5: Set Up GitHub Secrets¶

    Create Cloudflare API Token¶

    1. Go to Cloudflare API Tokens
    2. Click Create Token
    3. Use Custom token template:
    4. Permissions:
      • Account: Cloudflare Pages:Edit
      • Zone: Zone:Read
    5. Account Resources: Include → Your account

    6. Zone Resources: Include → Specific zone → kssocialistbookclub.com

    7. Copy the generated token

    Add Secrets to GitHub¶

    In your GitHub repository:

    1. Go to Settings → Secrets and variables → Actions
    2. Add new secrets:
    3. CLOUDFLARE_API_TOKEN: Your API token
    4. CLOUDFLARE_ACCOUNT_ID: Found in Cloudflare dashboard

    Step 6: Configure Giscus¶

    Update Origins¶

    Edit giscus.json to include your production domain:

    {
  "origins": [
    "https://druids.kssocialistbookclub.com",
    "http://localhost:8000",
    "http://127.0.0.1:8000"
  ],
  "originsRegex": [
    "https://.*\\.kssocialistbookclub\\.com"
  ]
}

    Update Comments Theme¶

    In overrides/partials/comments.html, update the theme URL:

    var theme = "https://druids.kssocialistbookclub.com/assets/stylesheets/giscus-druids.css";

    Step 7: Deployment Workflow¶

    Automatic Deployments¶

    Every push to main branch triggers:

    1. GitHub Actions builds the site
    2. Deploys to Cloudflare Pages
    3. Updates production site
    4. Invalidates cache

    Preview Deployments¶

    Pull requests create preview deployments:

    • Unique URL for each PR
    • Automatic comments with preview link
    • Isolated from production
    • Deleted when PR is closed

    Monitoring and Maintenance¶

    Build Status¶

    Monitor builds in:

    • Cloudflare Dashboard: Workers & Pages → Your project
    • GitHub Actions: Repository → Actions tab

    Analytics¶

    View site analytics:

    1. Cloudflare Dashboard → Analytics & Logs
    2. Web Analytics → Your domain
    3. No cookies or tracking required

    Performance¶

    Cloudflare Pages provides:

    • Global CDN distribution
    • Automatic caching
    • HTTP/3 support
    • Brotli compression
    • Image optimization

    Troubleshooting¶

    Build Failures¶

    # Test build locally
mkdocs build --strict

# Check Python version
python --version  # Should be 3.11+

# Verify dependencies
pip list

    Domain Issues¶

    # Check DNS propagation
dig druids.kssocialistbookclub.com

# Verify CNAME record
nslookup druids.kssocialistbookclub.com

    Common Errors¶

    Error Solution
    Build timeout Reduce site size or optimize images
    Module not found Add to requirements.txt
    DNS not resolving Wait for propagation (up to 48h)
    SSL errors Check domain configuration

    Advanced Configuration¶

    Build Optimization¶

    # mkdocs.yml
plugins:
  - minify:
      minify_html: true
      minify_js: true
      minify_css: true
  - optimize:
      cache: true
      optimize_pngs: true
      optimize_jpgs: true

    Redirects¶

    Create _redirects file in docs/:

    # Redirect old URLs
/old-page /new-page 301
/docs/* /:splat 301

    Headers¶

    Create _headers file in docs/:

    /*
  X-Frame-Options: DENY
  X-Content-Type-Options: nosniff
  Referrer-Policy: strict-origin-when-cross-origin

    Security Considerations¶

    Access Control¶

    For private documentation:

    1. Enable Cloudflare Access
    2. Configure authentication rules
    3. Set up team permissions

    Content Security¶

    • All deployments are immutable
    • Version history maintained
    • Rollback capability available
    • Automatic HTTPS enforcement
