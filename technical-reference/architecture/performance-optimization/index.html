    Performance Optimization Guide¶

    Strategies to keep the DRUIDS wiki fast and responsive as it grows.

    Build Performance¶

    Measuring Build Times¶

    # Time the build process
time mkdocs build

# Verbose output to identify slow steps
mkdocs build --verbose

# Profile plugin performance
PYTHONPROFILE=1 mkdocs build

    Optimization Strategies¶

    1. Conditional Plugin Loading¶

    Disable heavy plugins during development:

    # mkdocs.yml
plugins:
  - search
  - minify:
      enabled: !ENV [CI, false]  # Only in production
  - optimize:
      enabled: !ENV [CI, false]
  - privacy:
      enabled: !ENV [PRODUCTION, false]

    2. Use Plugin Caching¶

    Many plugins support caching:

    plugins:
  - optimize:
      cache: true
      cache_dir: .cache/optimize
  - privacy:
      cache: true

    3. Optimize Search Index¶

    For large sites:

    plugins:
  - search:
      prebuild_index: true  # Build at compile time
      min_search_length: 3  # Reduce index size

    Image Optimization¶

    Pre-Processing Images¶

    # Install tools
sudo apt-get install imagemagick jpegoptim optipng

# Batch optimize JPEGs
find docs/assets/images -name "*.jpg" -exec jpegoptim --strip-all {} \;

# Batch optimize PNGs
find docs/assets/images -name "*.png" -exec optipng -o7 {} \;

# Resize large images
mogrify -resize 1920x1920\> -quality 85 docs/assets/images/*.jpg

    Using WebP Format¶

    # Convert to WebP
for file in docs/assets/images/*.jpg; do
  cwebp -q 80 "$file" -o "${file%.jpg}.webp"
done

    In markdown:

    <picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Description">
</picture>

    CSS Optimization¶

    Minimize CSS¶

    Use the minify plugin:

    plugins:
  - minify:
      minify_css: true
      css_files:
        - assets/stylesheets/*.css

    Remove Unused CSS¶

    /* Before */
.unused-class { ... }
.old-component { ... }

/* After - Remove unused selectors */

    Use CSS Variables¶

    /* Instead of repeating colors */
:root {
  --primary: #7F8C8D;
}

.component { color: var(--primary); }

    JavaScript Optimization¶

    Lazy Load Heavy Scripts¶

    // Only load when needed
if (document.querySelector('.mermaid')) {
  import('./mermaid.min.js').then(module => {
    module.initialize();
  });
}

    Minimize JavaScript¶

    plugins:
  - minify:
      minify_js: true
      js_files:
        - assets/javascripts/*.js

    Page Load Optimization¶

    Critical CSS¶

    Inline critical CSS in overrides:

    <!-- overrides/partials/styles.html -->
<style>
  /* Critical CSS here */
  body { background: #0A0E27; }
  .md-header { /* ... */ }
</style>

    Lazy Loading¶

    For comments and non-critical features:

    <!-- Comments loaded on scroll -->
<div id="comments-placeholder">
  <button onclick="loadComments()">Load Comments</button>
</div>

<script>
function loadComments() {
  // Load Giscus script
}
</script>

    Resource Hints¶

    Add to overrides/main.html:

    <link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://giscus.app">
<link rel="dns-prefetch" href="https://cdn.jsdelivr.net">

    Content Optimization¶

    Split Large Pages¶

    Instead of one massive page:

    docs/
  complete-guide.md (10,000 lines)

    Split into sections:

    docs/guide/
  index.md
  getting-started.md
  advanced-topics.md
  troubleshooting.md

    Optimize Tables¶

    For large tables, use pagination:

    <div class="table-wrapper" data-page="1">
  <table>
    <!-- First 50 rows -->
  </table>
  <button onclick="loadMore()">Load More</button>
</div>

    Server Optimization¶

    Enable Compression¶

    Cloudflare Pages automatically compresses, but for local:

    # Custom server with compression
from livereload import Server
import gzip

server = Server()
server.watch('docs/', 'mkdocs build')
server.serve(root='site/', open_url=True)

    Cache Headers¶

    In _headers file:

    /*
  Cache-Control: public, max-age=3600

/assets/*
  Cache-Control: public, max-age=31536000

/*.html
  Cache-Control: public, max-age=300

    Monitoring Performance¶

    Lighthouse Scores¶

    Regular testing:

    # Install Lighthouse
npm install -g lighthouse

# Test performance
lighthouse https://druids.kssocialistbookclub.com --view

    Target scores:

    • Performance: > 90
    • Accessibility: > 95
    • Best Practices: > 90
    • SEO: > 90

    Real User Monitoring¶

    Add analytics to track actual performance:

    // Track page load time
window.addEventListener('load', function() {
  const loadTime = performance.timing.loadEventEnd - performance.timing.navigationStart;
  console.log('Page load time:', loadTime + 'ms');
});

    Plugin-Specific Optimizations¶

    Git Revision Plugin¶

    Limit history depth:

    plugins:
  - git-revision-date-localized:
      enable_creation_date: false  # Faster
      strict: false

    Mermaid Diagrams¶

    Load on demand:

    // Only initialize when diagrams exist
if (document.querySelector('.mermaid')) {
  mermaid.initialize({ startOnLoad: true });
}

    Search Optimization¶

    plugins:
  - search:
      lang: en
      separator: '[\s\-\.]+'  # Simpler regex
      min_search_length: 3
      prebuild_index: true

    Development Workflow¶

    Fast Iteration¶

    # Dirty reload - only rebuild changed files
mkdocs serve --dirtyreload

# Skip plugins during development
FAST_BUILD=1 mkdocs serve

    Parallel Builds¶

    For multi-site setups:

    # Build sites in parallel
mkdocs build -f mkdocs-public.yml &
mkdocs build -f mkdocs-member.yml &
wait

    Common Performance Issues¶

    Slow Search¶

    Problem: Search becomes sluggish with 1000+ pages

    Solution:

    plugins:
  - search:
      prebuild_index: true
      indexing: 'titles'  # Index only titles

    Large Git History¶

    Problem: Git plugins slow with long history

    Solution:

    # Shallow clone for CI/CD
git clone --depth 1 https://github.com/ks-sbc/DRUIDS.git

    Memory Issues¶

    Problem: Build runs out of memory

    Solution:

    # Increase Node memory
export NODE_OPTIONS="--max-old-space-size=4096"
mkdocs build

    Best Practices¶

    1. Regular Audits: Monthly performance checks
    2. Image Budget: Max 200KB per image
    3. Page Budget: Max 2MB per page total
    4. Plugin Audit: Remove unused plugins
    5. Content Review: Archive old content

    Remember: Performance is a feature. Fast sites keep members engaged!
