    MkDocs Technical Reference¶

    Overview¶

    MkDocs is a static site generator designed for building project documentation. DRUIDS uses MkDocs with the Material theme to create a searchable, mobile-friendly documentation website from our Markdown files.

    Key Learning: When facing broken custom themes, evaluate switching to proven solutions like Material theme with CSS overlays rather than debugging custom implementations. This approach delivers working functionality in minutes vs hours of troubleshooting.

    Architecture¶

    Core Components¶

    1. MkDocs Engine (v1.6+)
    2. Python-based static site generator
    3. Converts Markdown to HTML
    4. Built-in development server with hot reload
    5. Extensive plugin architecture

    6. Warning/error validation system

    7. Material Theme (v9.6+)

    8. Modern, responsive design with proven navigation
    9. Advanced features: tabs, sections, instant loading
    10. Built-in search with highlighting
    11. Dark/light mode with theme toggle

    12. Code copying, back-to-top, and mobile optimization

    13. Build Pipeline

    Markdown Files → MkDocs Engine → HTML/CSS/JS → Static Site
                     ↓
            Plugin Processing (search, git dates, etc.)
                     ↓
            Theme Application (Material + Custom CSS)
                     ↓
            Asset Optimization & Validation

    DRUIDS-Specific Architecture¶

    DRUIDS Documentation Stack:
┌─────────────────────────────────────────────────────────┐

    Configuration Schema¶

    Required Fields¶

    site_name: string        # Project name (required)

    Site Information¶

    site_description: string # Meta description
site_author: string     # Meta author
site_url: string        # Canonical URL (with trailing slash)
repo_url: string        # Repository link
repo_name: string       # Repository display name

    Directory Configuration¶

    docs_dir: path          # Source directory (default: 'docs')
site_dir: path          # Output directory (default: 'site')
use_directory_urls: bool # Clean URLs (default: true)
strict: bool            # Fail on warnings (default: false)
dev_addr: string        # Dev server (default: '127.0.0.1:8000')

    Plugin System¶

    Core Plugins¶

    1. search - Built-in full-text search
    plugins:
  - search:
      separator: '[\s\-\.]+'  # Token separator regex
      lang: [en]              # Language support
      prebuild_index: true    # Python/JavaScript index
    1. git-revision-date-localized - Show last modified dates
    - git-revision-date-localized:
    enable_creation_date: true
    type: iso_date           # date, datetime, iso_date, iso_datetime
    timezone: America/Chicago
    locale: en

    Material Theme Plugins¶

    1. social - Generate social media cards
    2. tags - Tag-based organization
    3. blog - Blog functionality
    4. offline - Offline documentation

    Theme Configuration¶

    Material Features (DRUIDS Configuration)¶

    Recommended Feature Set (Battle-tested for DRUIDS):

    theme:
  name: material
  features:
    # Essential Navigation (fixes dropdown issues)
    - navigation.tabs         # Top-level tabs with working dropdowns
    - navigation.sections     # Section grouping
    - navigation.expand       # Auto-expand navigation
    - navigation.path         # Breadcrumb navigation
    - navigation.top          # Back to top button

    # Search Enhancement
    - search.highlight        # Highlight search terms
    - search.share           # Shareable search URLs

    # Content Features
    - content.code.copy      # Copy buttons for code blocks

  # DRUIDS Revolutionary Color Scheme
  palette:
    - scheme: slate           # Dark theme primary
      primary: green          # Forest green
      accent: amber           # Earth tone accent
      toggle:
        icon: material/brightness-4
        name: Switch to light mode
    - scheme: default         # Light theme fallback
      primary: green
      accent: amber
      toggle:
        icon: material/brightness-7
        name: Switch to dark mode

  font:
    text: Inter             # Clean, readable font
    code: JetBrains Mono    # Monospace for code

    Features to Avoid (Can cause issues):

    • navigation.instant - Can break custom CSS
    • navigation.tracking - Unnecessary complexity
    • header.autohide - Conflicts with fixed headers

    Color Schemes¶

      palette:
    - media: "(prefers-color-scheme: light)"
      scheme: default

      accent: red       # Accent color
    - media: "(prefers-color-scheme: dark)"
      scheme: slate
      primary: red
      accent: amber

    Build Process¶

    Development Build¶

    mkdocs serve
# Starts development server with hot reload
# Watches: docs_dir, theme, mkdocs.yml

    Production Build¶

    mkdocs build
# Generates static site in site_dir
# Validates: internal links, missing files

    Build Options¶

    mkdocs build --clean     # Remove stale files
mkdocs build --strict    # Fail on warnings
mkdocs build --verbose   # Detailed output
mkdocs build --quiet     # Minimal output

    Advanced Features¶

    Environment Variables¶

    site_url: !ENV SITE_URL
analytics:
  property: !ENV [ANALYTICS_TOKEN, "default"]

site_name: DRUIDS Wiki  # Override base

def on_pre_build(config):

    return files

def on_nav(nav, config, files):
    """Process navigation"""
    return nav

    exclude_docs: |

    1. Optimize search index
    plugins:
  - search:
      min_search_length: 3
      prebuild_index: python
    1. Minimize assets
    plugins:
  - minify:
      minify_html: true
      minify_js: true

    1. Enable instant loading
    theme:
  features:
    - navigation.instant
    - navigation.instant.prefetch
    1. Lazy load images
    markdown_extensions:
  - attr_list
# Then use: <!-- ![Image](image.jpg) -->{: loading=lazy }

    Security Considerations¶

    Content Security¶

    1. Sanitize user content - MkDocs escapes HTML by default
    2. Validate external links - Use validation.links settings
    3. Restrict file access - Use exclude_docs patterns

    Build Security¶

    1. Pin dependencies - Use specific versions
    2. Validate plugins - Only use trusted sources
    3. Secure CI/CD - Protect deployment credentials

    Troubleshooting¶

    Common Issues¶

    1. Dropdown menus not working / all opening at once
    2. Root Cause: Custom theme navigation conflicts
    3. Solution: Switch to Material theme with proven navigation
    4. Quick Fix: Use navigation.tabs and navigation.sections features

    5. Avoid: Custom JavaScript for navigation - Material handles this

    6. Missing Python dependencies

    # Error: "cannot find module 'pymdownx'"
pip install mkdocs-material    # Includes pymdownx dependencies

# Error: "cannot find module 'materialx.emoji'"
# Fix emoji configuration:
markdown_extensions:
  - pymdownx.emoji:
      emoji_generator: !!python/name:pymdownx.emoji.to_svg
      emoji_index: !!python/name:pymdownx.emoji.twemoji
    1. Navigation not updating
    2. Clear browser cache
    3. Check YAML indentation (use 2 spaces)
    4. Validate file paths (case-sensitive)

    5. Restart dev server: Ctrl+C then mkdocs serve

    6. Search not working

    7. Rebuild search index: mkdocs build --clean
    8. Check JavaScript console for errors

    9. Verify search plugin enabled in mkdocs.yml

    10. Theme not loading

    11. Install theme: pip install mkdocs-material
    12. Check theme name spelling in mkdocs.yml

    13. Verify version compatibility (Material 9.6+ recommended)

    14. Custom CSS not applying

    15. Verify extra_css path in mkdocs.yml
    16. Check CSS file exists: docs/stylesheets/custom.css
    17. Clear browser cache and hard refresh

    18. Use browser dev tools to debug CSS specificity

    19. Broken links warnings

    WARNING - Doc file contains a link 'missing-file.md', but target not found
    • Fix: Update links to existing files
    • Suppress: Use mkdocs build without --strict
    • Validate: Regular link auditing during content updates

    DRUIDS Standardized Frontmatter:

    ---
title: "Document Title"

    Best Practices Learned¶

    Theme Selection Strategy¶

    1. Start with proven themes (Material) rather than custom development
    2. Use CSS overlays for visual customization
    3. Test navigation thoroughly - dropdowns are complex
    4. Prioritize functionality over perfect visual customization

    Content Organization¶

    1. Standardize frontmatter across all documentation
    2. Use consistent navigation structure (4-tier: Tutorials/How-To/Reference/Explanation)
    3. Maintain relative links for portability
    4. Regular link validation to prevent broken references

    Development Workflow¶

    1. Use external dev server for team access: --dev-addr=0.0.0.0:8001
    2. Validate builds strictly in CI/CD: mkdocs build --strict
    3. Clean builds for production: mkdocs build --clean
    4. Version pin dependencies for reproducible builds

    Performance Optimization¶

    1. Enable instant loading with Material theme features
    2. Optimize search with appropriate settings
    3. Use code copy buttons for better UX
    4. Implement back-to-top for long documents
