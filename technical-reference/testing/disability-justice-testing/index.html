    Disability Justice Testing Framework¶

    Overview¶

    Traditional test-driven development (TDD) often perpetuates ableism by centering non-disabled experiences and prioritizing compliance over actual usability. This framework reimagines testing through a disability justice lenswhere tests become tools for liberation rather than checkboxes for compliance.

    Core Principles¶

    1. Nothing About Us Without Us¶

    Testing must be done WITH disabled communities, not FOR them:

    //  Paternalistic Approach
test('form has ARIA labels', () => {
  expect(form.querySelector('[aria-label]')).toBeTruthy();
});

//  Community-Centered Approach
test('form is understandable via screen reader', async () => {
  const tester = await getCommunitytester('screen-reader-user');
  const result = await tester.evaluateForm(form);
  expect(result.canUnderstandPurpose).toBe(true);
  expect(result.canCompleteTask).toBe(true);
});

    2. Outcome Over Compliance¶

    Measure whether disabled users can accomplish tasks, not whether code meets specifications:

    describe('Task Completion Testing', () => {
  test('users with motor disabilities can submit timesheet', async () => {
    const user = createTestUser({ 
      disability: 'motor',
      assistiveTech: 'speech-recognition' 
    });

    const result = await user.completeTask('submit-timesheet');

    expect(result.completed).toBe(true);
    expect(result.painPoints).toHaveLength(0);
    expect(result.timeToComplete).toBeLessThan(user.baseline * 1.5);
  });
});

    3. Intersectional Coverage¶

    Disability doesn't exist in isolation. Test for multiply-marginalized experiences:

    const requiredPerspectives = [
  'blind-keyboard-user',
  'low-vision-magnification', 
  'motor-disability-speech',
  'cognitive-differences',
  'chronic-illness-fatigue',
  'deafblind-braille-user',
  'neurodivergent-sensory'
];

    Implementation Patterns¶

    Chronic Illness Accommodations¶

    describe('Energy Conservation Features', () => {
  test('all forms save progress automatically', () => {
    // Accommodates fatigue, brain fog, pain flares
    const form = renderComplexForm();
    fillPartialForm(form, 30); // 30% complete

    triggerAutoSave();
    const saved = getSavedFormData();

    expect(saved.percentComplete).toBe(30);
    expect(saved.canResumeAnytime).toBe(true);
  });

  test('generous timeout warnings', () => {
    const WARNING_TIME = getSessionTimeoutWarning();

    // Brain fog needs processing time
    expect(WARNING_TIME).toBeAtLeast(5 * MINUTES);
    expect(canRequestExtension()).toBe(true);
  });
});

    Cognitive Load Management¶

    describe('ADHD-Friendly Patterns', () => {
  test('focus persistence through distractions', () => {
    const element = document.activeElement;

    // Simulate notification/distraction
    triggerSystemNotification();

    // Focus shouldn't be stolen
    expect(document.activeElement).toBe(element);
    expect(getFocusIndicator(element).visible).toBe(true);
  });

  test('clear navigation hierarchy', () => {
    const hierarchy = getNavigationStructure();

    expect(hierarchy.maxDepth).toBeLessThanOrEqual(3);
    expect(hierarchy.hasCircularPaths).toBe(false);
    expect(hierarchy.breadcrumbsAlwaysVisible).toBe(true);
  });
});

    Beyond WCAG Minimums¶

    describe('Real Contrast Needs', () => {
  test('contrast boost beyond AA standards', () => {
    enableUserPreference('ultra-high-contrast');

    const pairs = getAllColorPairs();
    pairs.forEach(pair => {
      const ratio = getContrastRatio(pair.foreground, pair.background);

      // WCAG AA is 4.5:1, but many need 15:1+
      expect(ratio).toBeGreaterThan(15);

      // But not pure white on black (halation)
      expect(pair).not.toBePureWhiteOnBlack();
    });
  });
});

    Multi-Modal Task Completion¶

    describe('Multiple Paths to Success', () => {
  test('date entry supports diverse input methods', () => {
    const methods = getDateInputMethods();

    expect(methods).toIncludeAll([
      'textInput',           // "2025-07-01"
      'dropdownSelects',     // Month/Day/Year dropdowns
      'calendarWidget',      // Visual calendar
      'voiceInput',         // "July first 2025"
      'relativeDate'        // "tomorrow", "next Monday"
    ]);

    // Each method independently functional
    methods.forEach(method => {
      const result = enterDateUsing(method, '2025-07-01');
      expect(result.success).toBe(true);
    });
  });
});

    Community Validation Process¶

    Test Co-Creation¶

    # .druids/community-testing.yaml
test_development:
  participants:
    recruited_from: "disability-justice-orgs"
    compensation: "$150/hour"
    time_commitment: "flexible-crip-time"

  process:
    - identify_barriers: "What prevents you from using similar tools?"
    - design_tests: "What would prove this works for you?"
    - validate_implementation: "Does this meet your needs?"
    - iterate: "What's still broken?"

    Continuous Feedback Loop¶

    class CommunityFeedback {
  async collectBarriers() {
    const channels = [
      'in-app-form',
      'email-plaintext',
      'voice-recording',
      'video-report',
      'scheduled-interview'
    ];

    const feedback = await gatherFromAllChannels(channels);

    // Convert to actionable tests
    feedback.forEach(item => {
      if (!hasTestFor(item.barrier)) {
        createTest({
          description: item.barrier,
          reporter: item.user,
          compensation: 'standard-rate',
          maintainer: item.community
        });
      }
    });
  }
}

    Budget Allocation¶

    True accessibility requires resource commitment:

    const accessibilityBudget = {
  minimum: 0.20, // 20% of total budget

  allocation: {
    communityCompensation: 0.40,  // Paying testers/consultants
    toolsAndLicenses: 0.20,       // Screen readers, etc
    teamEducation: 0.20,          // Disability justice training
    implementation: 0.20          // Extra dev time
  }
};

    Anti-Patterns to Avoid¶

    Compliance Theater¶

    // Bad: Checking boxes without understanding
test('has alt text', () => {
  images.forEach(img => expect(img.alt).toBeTruthy());
});

    Automated-Only Testing¶

    // Bad: Assuming robots can detect human barriers
runAxeCore().then(results => {
  expect(results.violations).toHaveLength(0);
});

    Single-Disability Focus¶

    // Bad: Only testing one type of disability
testOnlyWithScreenReader(page);

    Speed Over Access¶

    // Bad: Accessibility as afterthought
if (hasTimeLeft()) {
  addAccessibilityMaybe();
}

    Implementation Checklist¶

    • Recruit paid community testers across disabilities
    • Create test patterns WITH community input
    • Allocate 20%+ budget to accessibility
    • Set up continuous feedback channels
    • Make accessibility tests blocking for deploy
    • Track outcome metrics, not compliance
    • Compensate all disability expertise
    • Document and share learnings publicly
    • Train team in disability justice principles
    • Celebrate access wins as revolutionary victories

    Example Test Suite¶

    // __tests__/disability-justice/voting-system.test.js
describe('Accessible Democratic Centralism', () => {
  const testers = getCommunityTesters([
    'screen-reader',
    'motor-disability',
    'cognitive-difference',
    'chronic-illness'
  ]);

  test.each(testers)('%s can participate in voting', async (tester) => {
    const session = await tester.startVotingSession();

    // Can understand the proposal
    expect(session.proposalClarity).toBeAtLeast(8/10);

    // Can make informed choice
    expect(session.hasAccessToInfo).toBe(true);
    expect(session.canAskQuestions).toBe(true);

    // Can submit vote privately
    expect(session.voteSubmitted).toBe(true);
    expect(session.privacyMaintained).toBe(true);

    // Receives confirmation
    expect(session.gotConfirmation).toBe(true);
  });
});
