    Getting Started with DRUIDS Aesthetics¶

    • TODO: Proof read this
    • TODO: Format correctly

    Overview¶

    This tutorial will guide you through implementing the DRUIDS aesthetic system in a new Obsidian vault. By the end, you'll have a fully themed vault with security visual coding.

    Prerequisites¶

    • Obsidian installed (v1.0 or higher)
    • Access to the DRUIDS shared repository
    • Basic understanding of CSS (helpful but not required)

    Lesson 1: Setting Up Your First Theme¶

    Step 1: Create the Snippets Directory¶

    First, we need to create a place for our CSS files:

    1. Open your Obsidian vault
    2. Go to Settings Appearance
    3. Click "Open snippets folder"
    4. This opens the .obsidian/snippets/ directory

    Step 2: Download Core Files¶

    Copy these files from the shared repository to your snippets folder:

    # From repos/shared/.obsidian/snippets/
00-core-variables.css
01-security-system.css
02-prairie-radical.css

    Step 3: Enable the Snippets¶

    1. Back in Obsidian Settings Appearance
    2. Scroll to "CSS snippets"
    3. Toggle on each file:
    4. 00-core-variables
    5. 01-security-system
    6. 02-prairie-radical

    Step 4: See Your Theme in Action¶

    Create a new note and add this content:

    # Welcome to DRUIDS

This is a public document using the Prairie Radical theme.

<div class="security-l0">
This content is marked as public (L0).
</div>

    You should see:

    • Warm parchment background
    • Playfair Display headers
    • Blue security indicator

    Lesson 2: Understanding Security Levels¶

    Step 1: Create Security Test Notes¶

    Create three notes to test each security level:

    Note 1: Public Content

    ---
security: L0
---

# Public Educational Material

<div class="security-indicator security-l0">
This content is freely available to everyone.
</div>

    Note 2: Member Content

    ---
security: L1
---

# Member Resources

<div class="security-indicator security-l1">
This content requires member access.
</div>

    Note 3: Cadre Content

    ---
security: L2
---

# Strategic Planning

<div class="security-indicator security-l2">
This content is highly restricted.
</div>

    Step 2: Observe the Visual Differences¶

    Each note should display:

    • Different colored left border (blue, amber, red)
    • Tinted background
    • Security level indicator

    Step 3: Add Security Badges¶

    Enhance your notes with badges:

    <span class="security-badge security-badge-l0">Public</span>

# Document Title

<span class="security-badge security-badge-l1">Member Only</span>

# Internal Documentation

<span class="security-badge security-badge-l2">Cadre Restricted</span>

# Sensitive Operations

    Lesson 3: Switching Themes¶

    Step 1: Enable Multiple Themes¶

    1. Copy additional theme files:
    2. 03-neobrutalist.css

    3. 04-mandalorian.css

    4. Enable only ONE theme at a time:

    5. For Prairie Radical: Enable 02-prairie-radical
    6. For Neobrutalist: Enable 03-neobrutalist
    7. For Mandalorian: Enable 04-mandalorian

    Step 2: Experience Each Theme¶

    Prairie Radical (Educational/Public):

    • Soft, warm colors
    • Classic typography
    • Approachable feeling

    Neobrutalist (Urgent/Action):

    • High contrast yellow/black
    • Bold borders and shadows
    • Aggressive, attention-grabbing

    Mandalorian (Technical/Internal):

    • Dark background
    • Green accents
    • Professional, secure feeling

    Step 3: Choose Based on Context¶

    • Teaching a workshop? Prairie Radical
    • Organizing urgent action? Neobrutalist
    • Writing technical docs? Mandalorian

    Lesson 4: Customizing Components¶

    Step 1: Create a Test Component Page¶

    Create a note called "Component Showcase" with:

    # Component Showcase

## Buttons

<button class="druids-btn druids-btn-primary">Primary Action</button>
<button class="druids-btn druids-btn-secondary">Secondary Action</button>

## Cards

<div class="druids-card">
  <div class="druids-card-header">
    <h3>Card Title</h3>
  </div>
  <div class="druids-card-body">
    This is a card component with our theme styling.
  </div>
</div>

## Alerts

<div class="druids-alert druids-alert-info">
  <strong>Info:</strong> This is an informational message.
</div>

<div class="druids-alert druids-alert-warning">
  <strong>Warning:</strong> This requires your attention.
</div>

    Step 2: See Theme Variations¶

    Switch between themes to see how components adapt:

    • Prairie Radical: Subtle, paper-like cards
    • Neobrutalist: Bold black borders, dramatic shadows
    • Mandalorian: Dark cards with metal texture

    Lesson 5: Creating Your First Custom Style¶

    Step 1: Create a Custom Snippet¶

    Create a new file: 99-my-customizations.css

    /* My Custom DRUIDS Styles */

/* Custom accent color */
:root {
  --my-custom-accent: #7B3F99; /* Purple */
}

/* Custom component */
.my-special-box {
  background-color: var(--my-custom-accent);
  color: var(--druids-paper-white);
  padding: var(--druids-space-md);
  border-radius: var(--druids-border-radius-md);
  margin: var(--druids-space-md) 0;
}

/* Override heading color in Prairie Radical */
.theme-prairie-radical h1 {
  color: var(--my-custom-accent);
}

    Step 2: Use Your Custom Styles¶

    In a note:

    # My Custom Styled Document

<div class="my-special-box">
This box uses my custom purple accent color!
</div>

    Practice Exercises¶

    Exercise 1: Security Awareness¶

    Create a document that uses all three security levels with proper visual indicators.

    Exercise 2: Theme Comparison¶

    Create the same content in three different notes, each using a different theme. Compare the emotional impact.

    Exercise 3: Component Library¶

    Build a reference note with all available components styled for your preferred theme.

    Exercise 4: Custom Theme¶

    Create a custom color variant of one of the base themes using CSS variables.

    Troubleshooting¶

    Styles Not Appearing?¶

    1. Check that CSS snippets are enabled in Settings
    2. Reload Obsidian (Ctrl/Cmd + R)
    3. Verify file names match exactly

    Conflicts Between Themes?¶

    • Only enable one theme file at a time
    • Core variables and security system should always be enabled

    Performance Issues?¶

    • Disable unused theme files
    • Keep custom CSS minimal
    • Use CSS variables instead of repeated values

    Next Steps¶

    Now that you understand the basics:

    1. Review the Components Guide for all available components
    2. Study the Color System for color values
    3. Follow the Theme Implementation Guide

    Summary¶

    You've learned to:

    • Install and enable DRUIDS theme CSS
    • Apply security visual coding
    • Switch between aesthetic themes
    • Use pre-built components
    • Create custom styles

    The DRUIDS aesthetic system is now at your disposal. Use it to create beautiful, security-conscious, and politically coherent interfaces!
