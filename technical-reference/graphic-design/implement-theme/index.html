    How to Implement a Custom Theme¶

    • TODO: Proof read this
    • TODO: Format correctly

    Overview¶

    This guide shows you how to create a custom theme that follows DRUIDS aesthetic principles while expressing your organization's unique identity.

    Prerequisites¶

    • Understanding of DRUIDS aesthetic system
    • Basic CSS knowledge
    • Access to the shared repository
    • Text editor

    Basic Implementation¶

    1. Create Theme File Structure¶

    Create a new CSS file following the naming convention:

    XX-theme-name.css

    Where XX is a number (05-99 for custom themes).

    2. Define Theme Variables¶

    Start with this template:

    /* ===========================
   Theme: [Your Theme Name]
   Description: [Brief description]
   Author: [Your name]
   Version: 1.0
   =========================== */

.theme-[your-theme-name] {
  /* === Primary Colors === */
  --druids-background-primary: #[hex];
  --druids-background-secondary: #[hex];
  --druids-background-tertiary: #[hex];

  --druids-text-primary: #[hex];
  --druids-text-secondary: #[hex];
  --druids-text-muted: #[hex];

  --druids-accent-primary: #[hex];
  --druids-accent-secondary: #[hex];

  --druids-border-color: #[hex];

  /* === Component Specific === */
  --druids-button-bg: var(--druids-accent-primary);
  --druids-button-text: var(--druids-paper-white);
  --druids-button-hover-bg: #[hex];

  --druids-card-bg: var(--druids-background-secondary);
  --druids-card-border: 1px solid var(--druids-border-color);

  /* === Typography === */
  --druids-heading-font: '[Font Name]', serif;
  --druids-heading-weight: 700;
  --druids-body-weight: 400;
}

    3. Apply Theme Class¶

    Add to your Obsidian vault's root element:

    /* Auto-apply theme */
body {
  @extend .theme-[your-theme-name];
}

/* Or use manually */
.theme-[your-theme-name] {
  /* Your styles */
}

    Advanced Customization¶

    Custom Security Colors¶

    Override default security colors for your theme:

    .theme-[your-theme-name] {
  /* Custom security palette */
  --druids-security-l0-color: #[hex];
  --druids-security-l0-bg: rgba([r], [g], [b], 0.05);

  --druids-security-l1-color: #[hex];
  --druids-security-l1-bg: rgba([r], [g], [b], 0.05);

  --druids-security-l2-color: #[hex];
  --druids-security-l2-bg: rgba([r], [g], [b], 0.05);
}

    Theme-Specific Components¶

    /* Custom button style */
.theme-[your-theme-name] .druids-btn-primary {
  background: linear-gradient(135deg, 
    var(--druids-accent-primary) 0%, 
    var(--druids-accent-secondary) 100%);
  border: none;
  box-shadow: 0 4px 15px rgba(0, 0, 0, 0.2);
}

/* Custom card effect */
.theme-[your-theme-name] .druids-card {
  position: relative;
  overflow: hidden;
}

.theme-[your-theme-name] .druids-card::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: radial-gradient(circle, 
    rgba(255,255,255,0.1) 0%, 
    transparent 70%);
  transform: rotate(45deg);
  pointer-events: none;
}

    Unique Visual Effects¶

    /* Texture overlay */
.theme-[your-theme-name]::before {
  content: '';
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;
  background-image: url('/path/to/texture.png');
  opacity: 0.02;
  pointer-events: none;
  z-index: 1;
}

/* Animation */
@keyframes theme-pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.95; }
}

.theme-[your-theme-name] .druids-accent-element {
  animation: theme-pulse 4s ease-in-out infinite;
}

    Theme Variants¶

    Dark Mode Support¶

    /* Automatic dark mode */
@media (prefers-color-scheme: dark) {
  .theme-[your-theme-name] {
    --druids-background-primary: #[dark-hex];
    --druids-text-primary: #[light-hex];
    /* Override other variables */
  }
}

/* Manual dark mode toggle */
.theme-[your-theme-name][data-theme="dark"] {
  /* Dark mode overrides */
}

    High Contrast Mode¶

    @media (prefers-contrast: high) {
  .theme-[your-theme-name] {
    --druids-text-primary: #000000;
    --druids-background-primary: #FFFFFF;
    --druids-border-width: 2px;
  }
}

    Integration Steps¶

    1. Test Your Theme¶

    Create a test document:

    ---
cssclass: theme-[your-theme-name]
---

# Theme Test Document

<div class="security-indicator security-l0">
Public content test
</div>

<button class="druids-btn druids-btn-primary">
Test Button
</button>

<div class="druids-card">
  <h3>Test Card</h3>
  <p>Card content goes here.</p>
</div>

    2. Validate Accessibility¶

    Check contrast ratios:

    // Add to console
function checkContrast(bg, fg) {
  // Contrast calculation logic
  return ratio >= 4.5 ? 'PASS' : 'FAIL';
}

checkContrast('#[bg-hex]', '#[fg-hex]');

    3. Document Your Theme¶

    Create a README for your theme:

    # [Theme Name] Theme

## Description
[What inspired this theme? What's its purpose?]

## Color Palette
- Background: #[hex] - [Description]
- Text: #[hex] - [Description]
- Accent: #[hex] - [Description]

## Usage
Best suited for [use cases].

## Screenshots
[Include screenshots of your theme in action]

    Best Practices¶

    Do's¶

    • ✅ Maintain WCAG AA contrast ratios
    • ✅ Test with all security levels
    • ✅ Include hover/focus states
    • ✅ Support responsive design
    • ✅ Document color choices

    Don'ts¶

    • ❌ Override security color meanings
    • ❌ Use too many colors (5-7 max)
    • ❌ Forget print styles
    • ❌ Ignore performance impact
    • ❌ Break existing components

  --druids-accent-primary: #e60000;

    Troubleshooting¶

    Theme Not Applying¶

    1. Check CSS file is enabled in Obsidian
    2. Verify class name matches exactly
    3. Ensure no syntax errors in CSS
    4. Check specificity conflicts

    Colors Look Wrong¶

    1. Validate hex codes
    2. Check color space (RGB vs HSL)
    3. Test on different monitors
    4. Verify opacity values

    Performance Issues¶

    1. Optimize images/textures
    2. Reduce animation complexity
    3. Minimize CSS calculations
    4. Use CSS variables efficiently

    Next Steps¶

    • Share your theme with the community
    • Create variations for different contexts
    • Build matching components
    • Develop a theme switcher

    Remember: Your theme should serve the political and practical needs of your organization while maintaining the security-first approach of DRUIDS.
