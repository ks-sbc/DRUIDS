    Color Usage Guidelines¶

    Do's¶

    • Use primary colors for main actions and branding
    • Apply security colors consistently across all interfaces
    • Maintain contrast ratios for all text/background combinations
    • Use semantic colors for status messages

    Don'ts¶

    • Don't use colors as the only means of conveying information
    • Don't mix theme palettes within a single interface
    • Don't use pure white (#FFFFFF) - use Paper White instead
    • Don't use security colors for non-security purposes

    Implementation Notes¶

    CSS Custom Properties¶

    All colors should be defined as CSS custom properties for easy theming:

    :root {
  /* Brand Colors */
  --color-revolutionary-red: #CC3C3A;
  --color-theory-black: #121212;
  --color-paper-white: #F8F7F6;
  --color-comrade-gold: #D6AA00;

  /* Security Colors */
  --color-security-l0: #3B82F6;
  --color-security-l1: #D6AA00;
  --color-security-l2: #CC3C3A;
}

    SCSS Variables (for Quartz)¶

    $revolutionary-red: #CC3C3A;
$theory-black: #121212;
$paper-white: #F8F7F6;
$comrade-gold: #D6AA00;
