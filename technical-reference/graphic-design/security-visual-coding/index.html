    Overview¶

    The DRUIDS security visual coding system provides immediate visual recognition of content security levels across all interfaces. This system uses multiple sensory channels to ensure security awareness is maintained at all times.

    Security Level Definitions¶

    L0 - Public (Blue)¶

    • Purpose: Content freely available to everyone
    • Color: #3B82F6 (Accessible Blue)
    • Icon: or
    • Use Cases: Educational materials, public statements, recruitment content

    L1 - Member (Amber/Gold)¶

    • Purpose: Internal content for verified members
    • Color: #D6AA00 (Comrade Gold)
    • Icon: or
    • Use Cases: Meeting notes, development docs, member resources

    L2 - Cadre (Red)¶

    • Purpose: Highly restricted operational content
    • Color: #CC3C3A (Revolutionary Red)
    • Icon: or
    • Use Cases: Strategic planning, sensitive operations

    JavaScript Enhancement¶

    // Security level detection
class SecurityVisualizer {
  constructor() {
    this.levels = {
      public: { level: 0, class: 'security-l0', icon: '' },
      member: { level: 1, class: 'security-l1', icon: '' },
      cadre: { level: 2, class: 'security-l2', icon: '' }
    };
  }

  // Apply security visualization to element
  applySecurityLevel(element, level) {
    const security = this.levels[level];
    if (!security) return;

    element.classList.add(security.class);
    element.setAttribute('data-security', `l${security.level}`);
    element.setAttribute('aria-label', `Security Level ${security.level}: ${level}`);
  }

  // Auto-detect from file path
  detectFromPath(path) {
    if (path.includes('/public/')) return 'public';
    if (path.includes('/member/')) return 'member';
    if (path.includes('/cadre/')) return 'cadre';
    return 'public'; // Default to lowest security
  }
}

    Security Transitions¶

    /* Smooth transitions between security levels */
.security-transition {
  transition: 
    background-color var(--druids-transition-normal),
    border-color var(--druids-transition-normal);
}

/* Warning animation for security changes */
@keyframes security-warning {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.5; }
}

.security-level-changing {
  animation: security-warning 1s ease-in-out 3;
}

    @media print {
  /* Ensure security indicators are visible in print */
  .security-l0::before {
    content: "[PUBLIC] ";
    color: var(--druids-security-l0-text);
  }

  .security-l1::before {
    content: "[MEMBER ONLY] ";
    color: var(--druids-security-l1-text);
  }

  .security-l2::before {
    content: "[CADRE RESTRICTED] ";
    color: var(--druids-security-l2-text);
    font-weight: bold;
  }

  /* Add security watermarks */
  .security-l1,
  .security-l2 {
    position: relative;
  }

  .security-l1::after,
  .security-l2::after {
    content: attr(data-security-level);
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%) rotate(-45deg);
    font-size: 4rem;
    opacity: 0.1;
    text-transform: uppercase;
  }
}
