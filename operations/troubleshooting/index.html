    Troubleshooting Guide¶

    Common issues and solutions for DRUIDS wiki operations.

    Build Issues¶

    MkDocs Build Fails¶

    Error: ERROR - Config value 'theme': Unrecognised theme name: 'material'

    Solution:

    pip install mkdocs-material
# Or install all dependencies
pip install -r requirements.txt

    Error: yaml.constructor.ConstructorError: could not determine a constructor

    This happens with Python-specific YAML tags. Solution:

    # Install the specific plugin causing the issue
pip install mkdocs-mermaid2-plugin

    Missing Modules¶

    Error: ModuleNotFoundError: No module named 'mkdocs_material'

    Solution:

    # Ensure you're in the virtual environment
source venv/bin/activate  # Linux/Mac
# or
venv\Scripts\activate  # Windows

# Install requirements
pip install -r requirements.txt

    Giscus Comments Issues¶

    Comments Not Showing¶

    1. Check frontmatter:
    ---
comments: true
---
    1. Verify GitHub Discussions is enabled:
    2. Go to repository Settings → Features

    3. Enable "Discussions"

    4. Check browser console for errors:

    5. Press F12
    6. Look for CORS or authentication errors

    Wrong Theme Loading¶

    If comments appear with wrong colors:

    1. Clear browser cache
    2. Check theme URL in comments.html matches your domain
    3. Verify giscus-druids.css is being served

    Deployment Issues¶

    Cloudflare Pages Build Timeout¶

    Error: Build exceeded 20 minute timeout

    Solutions:

    1. Optimize images before committing
    2. Reduce number of plugins
    3. Use build caching:
    - uses: actions/cache@v4
  with:
    path: .cache
    key: mkdocs-${{ hashFiles('requirements.txt') }}

    Domain Not Resolving¶

    After adding custom domain:

    1. Wait up to 48 hours for DNS propagation
    2. Check DNS records:
    dig druids.kssocialistbookclub.com
nslookup druids.kssocialistbookclub.com
    1. Ensure CNAME record exists in Cloudflare DNS

    Git Issues¶

    Large File Errors¶

    Error: this exceeds GitHub's file size limit of 100.00 MB

    Solution:

    # Remove file from history
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch path/to/large-file' \
  --prune-empty --tag-name-filter cat -- --all

# Use Git LFS for large files
git lfs track "*.pdf"
git add .gitattributes

    Merge Conflicts in mkdocs.yml¶

    Common when multiple people edit navigation:

    # Accept both changes
git checkout --theirs mkdocs.yml
# OR
git checkout --ours mkdocs.yml

# Then manually merge navigation sections

    Theme Issues¶

    CSS Not Updating¶

    Browser caching issue. Solutions:

    1. Hard refresh: Ctrl+Shift+R (Cmd+Shift+R on Mac)
    2. Add cache busting to CSS:
    extra_css:
  - stylesheets/druids-material.css?v=1.1
    1. Clear MkDocs cache:
    rm -rf .cache/
mkdocs build --clean

    Dark Mode Not Working¶

    Check Material theme configuration:

    theme:
  palette:
    scheme: slate  # Must be 'slate' for dark mode

    Search Issues¶

    Search Not Finding Content¶

    1. Rebuild search index:
    mkdocs build --clean
    1. Check search configuration:
    plugins:
  - search:
      lang: en
      separator: '[\s\-\.\(\)]+'
    1. Verify content is indexed:
    2. Check for exclude patterns
    3. Ensure pages aren't draft

    Search Too Slow¶

    For large sites:

    plugins:
  - search:
      prebuild_index: true  # Build index at build time

    Plugin Conflicts¶

    Multiple Plugins Processing Same Content¶

    Order matters in mkdocs.yml:

    plugins:
  - search        # Process first
  - minify        # Process last

    Mermaid Diagrams Not Rendering¶

    Ensure proper configuration:

    markdown_extensions:
  - pymdownx.superfences:
      custom_fences:
        - name: mermaid
          class: mermaid
          format: !!python/name:mermaid2.fence_mermaid

    Performance Issues¶

    Slow Local Development¶

    1. Disable plugins in development:
    plugins:
  - search
  - minify:
      enabled: !ENV [CI, false]
    1. Limit file watching:
    mkdocs serve --dirtyreload

    Large Site Build Times¶

    1. Use caching:
    plugins:
  - optimize:
      cache: true
      cache_dir: .cache/optimize
    1. Optimize images before committing:
    # Install imagemagick
mogrify -resize 1920x1920\> -quality 85 docs/assets/images/*.jpg

    Common Error Messages¶

    Error Cause Solution
    Unrecognised configuration name Typo in mkdocs.yml Check spelling and indentation
    contains a link, but the target is not found Broken internal link Fix the link path
    failed to load external entity Network issue Check internet connection
    Permission denied File permissions Run with proper permissions

    Getting Help¶

    If these solutions don't work:

    1. Check logs carefully - The error message usually hints at the solution
    2. Search existing issues - GitHub Issues
    3. Ask in discussions - GitHub Discussions
    4. Review documentation - MkDocs Material Docs

    Remember: Most issues are configuration problems, not bugs. Double-check your setup first!
